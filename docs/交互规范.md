# ClipHub 交互规范

## 1. 页面结构

悬浮窗分为：

1. 拖动标题栏；
2. 搜索区；
3. 快速类型过滤；
4. 剪贴板列表；
5. 新增入口；
6. 编辑、筛选和标签等二级页面。

全部页面必须纳入统一返回栈和统一 UI 关闭链，不允许只依赖页面内关闭按钮。

## 2. 标题栏

包含：

- 拖动手柄；
- 标题“全局剪切板”；
- 置顶；
- 收起；
- 关闭。

### 拖动规则

- 标题栏空白区域可拖动；
- 按钮命中区域只执行按钮操作；
- 按下后移动超过阈值才进入拖动；
- 拖动结束后保存位置；
- 位置必须限制在安全显示区域；
- 窗口不能完全移出屏幕。

## 3. 置顶

“置顶”指悬浮窗保持显示，不是剪贴板项目置顶。

状态建议：

- 未置顶：复制或打开目标内容后可以按设置自动收起；
- 已置顶：保持当前窗口；
- 图标必须清楚区分两种状态；
- 状态需要持久化。

剪贴板项目置顶应使用卡片内独立操作，避免概念混淆。

## 4. 搜索栏

- 输入后延迟触发查询；
- 清空按钮仅在有文字时显示；
- 搜索时保留当前过滤条件；
- 搜索结果数量可在筛选入口附近显示；
- 关闭再打开时是否保留搜索词由设置决定。

## 5. 来源应用过滤

应用列表至少显示：

- 应用图标；
- 应用名；
- 包名；
- 当前记录数量。

支持：

- 单选；
- 多选预留；
- “来源未知”；
- 快速清除。

## 6. 类型过滤

首版提供：

- 全部；
- 文本；
- 网址；
- 电话。

选中状态必须同时通过颜色、描边或图标表达，不能只依赖细微色差。

## 7. 剪贴板卡片

建议内容：

```text
拖动手柄
来源应用图标
来源应用名称
复制时间
内容预览
识别类型
自定义标签
置顶状态
更多操作
```

### 点击行为

- 单击卡片：复制原始内容；
- 点击内容区域：可配置为复制或打开详情；
- 长按：进入多选模式；
- 拖动手柄：调整顺序；
- 更多按钮：编辑、翻译、标签、删除。

为保持卡片紧凑，首版不建议同时常驻过多文字按钮。

## 8. 内容预览

- 最多显示若干行；
- 超长文本截断但不修改原始内容；
- 换行应正常展示；
- 命令片段可使用等宽字体；
- Emoji 必须能够正常显示；
- URL 与电话可使用强调样式；
- 强调样式不能影响文本复制结果。

## 9. 编辑器

字段：

- 内容；
- 内容类型；
- 标签；
- 置顶；
- 保存；
- 取消。

规则：

- 内容为空时不允许保存；
- 离开前存在未保存修改时提示；
- 保存按钮防重复点击；
- 编辑器关闭后恢复主窗口焦点和触摸状态；
- 软键盘弹出时确保操作按钮可见。

## 10. 新增

新增项目与编辑共用页面，但需要区分：

- 新增没有来源应用；
- 创建时间为保存时间；
- 可以直接标记标签；
- 保存后是否写入系统剪贴板由用户选择。

## 11. 翻译

- 原文只读展示；
- 译文与原文分离；
- 支持复制译文；
- 保存译文时创建新项目或覆盖当前内容必须明确选择；
- 网络失败不影响原记录；
- 翻译配置不在列表卡片中展开。

## 12. 删除

建议优先使用：

```text
已删除，可撤销
```

而不是每次弹出确认框。

以下情况需要确认：

- 批量删除；
- 清空全部；
- 删除标签并解除大量关联；
- 永久清理回收记录。

## 13. 收起状态

收起后可显示：

- 一个紧凑胶囊；
- 剪贴板图标；
- 未读或新增数量；
- 最近内容的简短提示，可由隐私设置关闭。

收起入口仍可拖动，但需要与点击展开区分。

## 14. 空状态

### 无记录

```text
尚无剪贴板记录
复制一段文本后会显示在这里
```

### 搜索无结果

```text
没有匹配“关键词”的记录
```

### 过滤无结果

```text
当前过滤条件下没有记录
清除过滤条件
```

### 错误状态

```text
剪贴板记录加载失败
重试
```

## 15. 亮暗色

- 使用统一主题模块；
- 文字与背景保证足够对比；
- 删除等危险操作不能只依赖红色；
- 标签颜色需要经过明暗色对比度修正；
- 系统主题变化时单次刷新，不重复重建窗口。

## 16. 动画

允许：

- 打开与关闭淡入淡出；
- 收起与展开尺寸过渡；
- 新增项目轻量进入；
- 删除项目收缩；
- 拖动排序位移动画。

限制：

- 无状态变化时停帧；
- 不使用持续空转动画；
- 动画结束后清理回调；
- WindowManager 移除前终止仍在运行的动画；
- 搜索输入时不对整个窗口执行动画。

## 17. 系统手势与 UI 退出

本节属于首版必要功能和发布阻断条件。

### 17.1 返回层级

系统返回优先级固定为：

```text
筛选面板 / 编辑器 / 详情页 / 标签管理
                  ↓
关闭当前最上层页面
                  ↓
返回剪贴板首页
                  ↓
首页再次返回
                  ↓
关闭全部 ClipHub UI
```

要求：

- Android 侧滑返回、预测性返回、三键导航返回和实体返回键行为一致；
- 当前存在多个次级窗口时，只关闭最上层窗口；
- 首页收到返回时，不再收起为胶囊，而是退出全部 UI；
- UI 退出不等于停止 ClipHub 后台实例；
- 剪贴板监听、数据库和正式控制入口必须继续运行。

### 17.2 系统返回接入方式

- 优先接入 Android 系统返回回调；
- Android 13 及以上需要兼容 `OnBackInvokedDispatcher`；
- 支持预测性返回时，返回动画进度只能作用于当前最上层页面；
- 不允许在屏幕左右边缘覆盖透明触摸层模拟侧滑返回；
- 不允许为了截获返回手势破坏系统导航区域、列表滚动或窗口拖动；
- 页面内返回按钮必须复用同一返回分发逻辑，不能维护另一套关闭顺序。

### 17.3 底部上拉进入最近任务

用户从底部上拉进入最近任务、切换应用或使 ClipHub UI 转入后台时，必须执行完整 UI 隐藏：

1. 关闭筛选面板；
2. 关闭编辑器；
3. 关闭详情页；
4. 关闭标签管理等其他次级窗口；
5. 关闭主列表窗口；
6. 移除全部遮暗层、辅助层和临时触摸层；
7. 保持剪贴板监听、SQLite、运行锁和控制入口。

下次通过“打开全局剪贴板”或“全局剪贴板开关”任务显示 UI 时，从首页开始，不恢复到上一次次级页面。

### 17.4 未保存编辑内容

- 侧滑返回离开编辑页时，按照普通编辑返回规则处理未保存修改；
- 底部上拉进入最近任务时不得弹出会阻塞系统手势的确认窗口；
- 进入后台不得自动把未保存内容写入正式剪贴板记录；
- 临时草稿只能保存在运行内存中，不保证 ClipHub 实例停止后仍可恢复。

### 17.5 统一关闭链

以下入口必须进入同一套幂等 UI 清理链：

- 系统侧滑返回；
- 三键返回键；
- 页面返回按钮；
- 页面关闭按钮；
- 首页关闭按钮；
- 底部上拉进入最近任务；
- `hide` 控制命令；
- `toggle` 在 UI 已显示时的关闭分支；
- ClipHub 停止前的 UI 清理。

关闭链必须满足：

- 最上层到最下层反向关闭；
- 同一个 View 最多移除一次；
- 快速连续返回不会发生重复 `removeView` 异常；
- 输入法在编辑器移除前正确隐藏；
- 关闭完成后所有窗口状态与实际附着状态一致；
- 不残留透明遮罩、遮暗背景或不可触摸区域。

### 17.6 必测场景

1. 首页侧滑返回后全部 UI 消失；
2. 详情页侧滑返回首页，首页再次侧滑退出；
3. 编辑页侧滑返回时正确处理未保存修改；
4. 筛选页侧滑返回首页；
5. 三键返回和侧滑返回顺序一致；
6. 底部上拉进入最近任务后全部 UI 消失；
7. 进入最近任务后复制新文本仍能入库；
8. 再次打开 UI 时从首页开始；
9. 快速连续返回不会崩溃或重复移除窗口；
10. 关闭后屏幕不存在透明触摸遮罩。
