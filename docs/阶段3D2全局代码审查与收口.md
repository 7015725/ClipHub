## 2. 第一轮审查结论

### 2.1 生命周期保护正常

Editor v11 与 Settings v10 均已具备：

- 延迟任务代际失效；
- 页面关闭时停止 IME polling；
- 移除 GlobalLayoutListener；
- WindowManager、Context、View 空值检查；
- 回调异常拦截，禁止异常逃逸到 system_server 主线程。

048 与 050 的真机结果证明关闭、stop、正式实例恢复和数据库清理均正常。

### 2.2 确定的性能收口项

Editor 和 Settings 当前在页面整个打开期间固定每 90ms 轮询 IME。

050 在约 18 秒内记录：

```text
imePollCount=173
```

键盘显示或输入框有焦点时保持 90ms，以保证输入法动画和面板位置响应；键盘隐藏且没有输入焦点时改为 420ms。GlobalLayoutListener 仍作为即时变化主通道，轮询继续作为 ColorOS / WindowInsets 异常时的兜底。

### 2.3 确定的回调一致性收口项

仍有少量 View.post() 未经过统一生命周期包装：

- Editor 长文本滚动到底部；
- Settings 当前输入框布局后滚动；
- Settings 标签分组布局后定位。

现有回调大多已有局部检查或 try/catch，但审查要求统一纳入：

- generation 校验；
- ready / attached 校验；
- pending / run / cancel / error 统计；
- shutdown 后安全取消。

### 2.4 暂不处理项

以下内容保留，不在本轮抽象：

- Editor 与 Settings 重复的 IME Insets 读取代码；
- 探测使用的详细状态字段；
- sectionAnchorSpacer 与 imeAnchorSpacer；
- 旧探测文件及历史失败探测。

原因：当前 Rhino ES5 模块之间没有稳定的共享 UI 生命周期层。过早抽象会扩大改动面，并可能破坏已经通过的 048～050 真机边界。
