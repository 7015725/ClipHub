# ClipHub 新增、编辑与置顶探测 017

## 1. 目的

`cliphub_editor_pin_probe_017.js` 验证阶段 3C1 的基础管理链路：

- 主列表新增入口；
- 独立可聚焦编辑窗口；
- 手动新增文本记录；
- 编辑已有正文；
- 置顶记录并调整排序；
- 新增、编辑和置顶后的列表刷新；
- 活动筛选在编辑后重新应用；
- 取消编辑不写入数据库；
- 编辑窗口在 App.stop 时完整清理；
- 新增、编辑和置顶状态跨重启持久化。

## 2. 当前版本

```text
moduleSetVersion=20260721.16
probeVersion=1
```

## 3. 功能边界

本阶段只实现：

```text
新增文本
编辑正文
置顶 / 取消置顶接口
```

不包含：

```text
拖动排序
自定义标签
长文本详情页
内容类型手动修改
来源信息手动修改
```

编辑已有记录时只修改 `content`，Repository 会同步重新计算 `normalized_hash`。来源、内容类型、敏感状态和复制次数保持不变。

新增记录使用：

```text
content_type=text
source_label=ClipHub 手动
source_package=null
source_uid=当前 ShortX 进程 UID
source_confidence=100
```

## 4. 隐私与事件边界

新增和编辑复用现有事件：

```text
clipboard_added
clipboard_merged
```

事件只包含：

```text
id
manual
mutation
pinned（仅置顶变化）
threadId
threadName
```

事件不包含剪贴板正文。

## 5. UI 行为

主列表顶部新增“新增”按钮。

每张记录卡片增加：

```text
置顶 / 取消置顶
编辑
删除
```

新增和编辑使用独立的 `TYPE_APPLICATION_OVERLAY` 窗口：

- 原生多行 EditText；
- 可获取输入焦点；
- 支持系统输入法；
- 保存和取消按钮；
- `SOFT_INPUT_ADJUST_RESIZE`；
- add/remove 和控件事件均在 Android `main` 线程；
- App.stop 时调用 `removeViewImmediate()` 清理。

## 6. 隔离边界

017 使用：

```text
shortx.getShortXDir()/ClipHubProbe017/
```

不会修改正式数据库，也不会读写系统剪贴板。

## 7. 执行顺序

1. 执行 `cliphub_stop_formal.js`；
2. 执行完整 `ClipHub.js`，确认同步到 `20260721.16`；
3. 执行 `cliphub_editor_pin_probe_017.js`。

入口成功后不要再次停止正式实例，017 会自行完成停止、隔离测试和恢复。

## 8. 探测过程

1. 写入两条隔离种子记录；
2. 显示主列表；
3. 自动点击“新增”；
4. 写入一条手动记录并保存；
5. 自动打开该记录并编辑；
6. 将较旧记录置顶并验证移动到首位；
7. 启用关键词筛选；
8. 修改匹配记录，使其退出筛选结果；
9. 重置筛选；
10. 打开新增窗口并取消，验证没有插入；
11. 再次打开编辑窗口后直接停止 App；
12. 验证编辑窗口自动清理；
13. 使用同一隔离数据库重启；
14. 验证新增、最终正文和置顶状态持久化；
15. 恢复正式实例并清理隔离目录。

报告不包含三条测试正文。
