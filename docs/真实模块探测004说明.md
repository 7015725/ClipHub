# ClipHub 真实模块探测 004

## 1. 目的

004 验证由 ShortX 入口自动下载到本地的真实模块：

- 本地 manifest 存在；
- 15 个模块数量完整；
- 每个模块的 Git blob SHA 与 manifest 一致；
- schema v1 正常迁移；
- Repository 插入、读取、更新、软删除和恢复；
- SHA-256 摘要随内容更新；
- 标签创建、关联和解除；
- 数据库事务包装层能够回滚；
- `ClipHub.App.stop()` 关闭数据库并释放文件锁；
- 停止后可以再次启动。

004 不再依赖运行目录中存在 `ClipHub.js` 副本，也不要求 Termux 安装模块。

## 2. 前置条件

先在 ShortX 中建立一个 Rhino JavaScript 任务，将仓库中的完整 `ClipHub.js` 粘贴进去并执行。

第一次执行时，入口会自动创建：

```text
shortx.getShortXDir()/ClipHub/
├── modules/
│   ├── ch_01_base.js
│   └── ...
└── cache/
    └── module-manifest.local.json
```

入口返回结果应至少满足：

```text
ok = true
started = true
entryVersion = 3
sync.downloadedCount = 15    # 首次运行
sync.updated = true          # 首次运行
sync.moduleSetVersion 非空
```

再次执行时，模块没有变化则通常为：

```text
sync.downloadedCount = 0
sync.updated = false
```

## 3. 数据隔离

004 使用运行目录：

```text
shortx.getShortXDir()/ClipHubProbe004/
```

模块来源为入口下载目录：

```text
shortx.getShortXDir()/ClipHub/modules/
```

本地 manifest：

```text
shortx.getShortXDir()/ClipHub/cache/module-manifest.local.json
```

004 不打开、不修改：

```text
shortx.getShortXDir()/ClipHub/data/cliphub.db
```

探测结束后会删除 `ClipHubProbe004` 目录。

## 4. 获取探测代码

在 Termux 中仅用于同步仓库和复制探测脚本，不用于安装模块：

```sh
cd ~/projects/github/ClipHub
git fetch origin
git switch agent/initialize-project-skeleton
git pull --ff-only

cp probes/cliphub_module_probe_004.js \
  "/storage/emulated/0/Download/ClipHub_真实模块探测_004.js"
```

在 ShortX 中建立另一个 Rhino JavaScript 单次任务，粘贴 004 完整代码并执行。
