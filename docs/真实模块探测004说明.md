# ClipHub 真实模块探测 004

## 1. 目的

004 使用仓库中的真实入口和真实模块验证：

- `ClipHub.js` 按固定顺序加载 15 个模块；
- schema v1 正常迁移；
- Repository 插入、读取、更新、软删除和恢复；
- SHA-256 摘要随内容更新；
- 标签创建、关联和解除；
- 数据库事务包装层能够回滚；
- `ClipHub.App.stop()` 关闭数据库并释放文件锁；
- 停止后可以再次启动。

## 2. 数据隔离

004 使用运行目录：

```text
shortx.getShortXDir()/ClipHubProbe004/
```

模块来源仍为：

```text
shortx.getShortXDir()/ClipHub/modules/
```

004 不打开、不修改：

```text
shortx.getShortXDir()/ClipHub/data/cliphub.db
```

探测结束后会删除 `ClipHubProbe004` 目录。

## 3. 安装真实运行文件

在 Termux 仓库目录执行：

```sh
cd ~/projects/github/ClipHub
git fetch origin
git switch agent/initialize-project-skeleton
git pull --ff-only
chmod +x scripts/install_shortx_runtime.sh
./scripts/install_shortx_runtime.sh \
  /data/system/shortx_OYdazjKnxMzhQykL
```

安装器只替换：

```text
ClipHub/ClipHub.js
ClipHub/modules/ch_*.js
```

不会执行 ClipHub，也不会创建正式数据库。

## 4. 执行探测

```sh
cp probes/cliphub_module_probe_004.js \
  "/storage/emulated/0/Download/ClipHub_真实模块探测_004.js"
```

在 ShortX 中建立 Rhino JavaScript 单次任务，粘贴完整代码并执行。
