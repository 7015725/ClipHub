# ClipHub 真实模块探测 004

## 1. 目的

004 验证入口自动下载后的真实模块：

- 本地 manifest 存在；
- 15 个模块均已下载；
- 每个模块 Git blob SHA 与 manifest 一致；
- schema v1 正常迁移；
- Repository 插入、读取、更新、软删除和恢复；
- SHA-256 摘要随内容更新；
- 标签创建、关联和解除；
- 数据库事务包装层能够回滚；
- `ClipHub.App.stop()` 关闭数据库并释放文件锁；
- 停止后可以再次启动。

## 2. 前置条件

先在 ShortX JavaScript 任务中执行最新版完整 `ClipHub.js`。

入口版本必须至少为：

```text
4
```

入口通过 `raw.githubusercontent.com` 下载：

```text
module-manifest.json
src/ch_01_base.js
...
src/ch_15_app.js
```

成功后运行目录应包含：

```text
shortx.getShortXDir()/ClipHub/
├── modules/
└── cache/module-manifest.local.json
```

不需要使用 Termux 安装模块。

## 3. 数据隔离

004 使用运行目录：

```text
shortx.getShortXDir()/ClipHubProbe004/
```

模块来源为入口已经下载的：

```text
shortx.getShortXDir()/ClipHub/modules/
```

004 不打开、不修改：

```text
shortx.getShortXDir()/ClipHub/data/cliphub.db
```

探测结束后会删除 `ClipHubProbe004` 目录。

## 4. 执行探测

在 Termux 仓库目录同步当前分支：

```sh
cd ~/projects/github/ClipHub
git fetch origin
git switch agent/initialize-project-skeleton
git pull --ff-only
```

复制探测脚本：

```sh
cp probes/cliphub_module_probe_004.js \
  "/storage/emulated/0/Download/ClipHub_真实模块探测_004.js"
```

在 ShortX 中建立 Rhino JavaScript 单次任务，粘贴完整代码并执行。
