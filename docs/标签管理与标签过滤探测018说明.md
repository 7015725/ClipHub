# ClipHub 标签管理与标签过滤探测 018

## 目的

验证模块集 `20260721.17` 的标签管理完整链路：

- 标签创建、名称归一化、重命名和删除；
- 标签与剪贴板记录的多对多绑定；
- 主列表标签摘要与“标签”入口；
- 标签单选、多选 OR 过滤；
- 标签与关键词之间的 AND 组合过滤；
- 活动标签筛选在绑定变化后自动重新应用；
- 标签事件不携带正文；
- 标签和绑定关系跨重启持久化；
- 删除标签不会删除原剪贴板记录。

## 版本

```text
moduleSetVersion=20260721.17
probeVersion=2
```

数据库继续使用 schema v2。`tags` 和 `clipboard_item_tags` 表自 schema v1 已存在，本阶段不执行数据库迁移。

### v2 修正

探测 v1 错误调用了不存在的 `ClipHub.Database.getSchemaVersion()`。数据库模块实际公开接口为 `ClipHub.Database.getVersion()`，v2 已修正该调用。

该问题只影响探测脚本，不影响模块集 `20260721.17`、正式数据库或标签功能。v1 异常退出后会停止隔离实例、恢复正式实例并删除隔离目录。

## 标签规则

- 标签名称去除首尾空格；
- 连续空白折叠为一个空格；
- 规范化名称不区分大小写；
- 标签名称最长 40 个字符；
- 同一记录与同一标签只能绑定一次；
- 删除标签只删除标签及关联关系，不删除剪贴板记录；
- 标签多选采用 OR；
- 标签与关键词、来源和内容类型之间采用 AND。

## UI 行为

每张历史卡片新增“标签”按钮。已绑定标签在正文下方显示为 `#标签名` 摘要。

标签管理复用独立可聚焦编辑窗口，包含：

- 标签名称 EditText；
- 新增标签按钮；
- 标签绑定/解绑按钮；
- 删除标签按钮；
- 系统输入法适配；
- App.stop 自动清理。

搜索与筛选面板新增“自定义标签”芯片区域。

## 隐私边界

`tags_changed` 事件只包含：

```text
action
itemId
tagId
threadId
threadName
```

事件和探测报告均不包含剪贴板正文或标签名称。

## 隔离范围

探测使用：

```text
shortx.getShortXDir()/ClipHubProbe018/
```

不会修改正式数据库，也不会读取或替换系统剪贴板。

## 执行顺序

### 首次从旧模块集执行

1. 执行 `cliphub_stop_formal.js`；
2. 执行完整 `ClipHub.js`，确认同步到 `20260721.17`；
3. 执行 `cliphub_tag_probe_018.js`。

### 已安装 `.17` 且正式实例正在运行

只需拉取最新分支、重新复制探测文件，然后直接执行探测 v2。探测会自行停止并恢复正式实例，不需要再次执行入口更新。
