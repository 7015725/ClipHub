# ClipHub 系统手势与后台退出探测 030 说明

## 1. 目标

探测模块集 `20260722.24` 的必要功能：

- Android 系统侧滑返回；
- Android 13 `OnBackInvokedCallback`；
- Android 14 `OnBackAnimationCallback`；
- 三键返回后备链；
- 次级页面返回首页；
- 首页返回关闭全部 ClipHub UI；
- 底部上拉进入最近任务或切换后台后关闭全部 UI；
- UI 关闭后后台实例、数据库和剪贴板监听继续运行；
- 关闭后清理残留 WindowManager View 和系统返回回调引用。

## 2. 已发现并修正的问题

### 030 v2：返回去重和回调清理

模块集 `20260722.23` 的结果中：

```text
editorBackHandled=true
editorClosedAfterBack=false
homeBackHandled=true
homeClosedAfterBack=false
duplicateBackCount=2
```

原因是返回分发器只使用全局 `180ms` 时间锁，不区分页面。详情、编辑、筛选和首页的合法连续返回被错误当成重复返回。

同一结果还显示关闭后 `registeredRootCount=5`，说明已失效窗口的系统返回注册项未在统一 UI 隐藏链中完全清理。

模块集 `20260722.24` 已修正：

- 真实系统返回使用统一 `system` 签名去重；
- 程序化返回按页面所有者去重；
- 关闭链最终扫描并移除残留 ClipHub WindowManager View；
- 注销失效的系统返回、焦点和附着监听；
- `Navigation.getState()` 自动剔除已分离窗口。

### 030 v3：详情测试夹具不足

030 v3 已确认模块集 `.24` 的核心修复生效：

```text
editorClosedAfterBack=true
filterClosedAfterBack=true
homeClosedAfterBack=true
callbacksClearedAfterHomeBack=true
callbacksClearedAfterSyntheticRecents=true
duplicateBackCount=0
registeredRootCount=0
```

但 v3 插入的测试文本没有达到主列表的长文本详情阈值：

```text
longItemCount=0
detailButtonCount=0
detailOpen=false
```

因此该次 `ok=false` 属于探测数据夹具错误，不是系统返回实现回退。由于详情没有打开，随后标记为 `detail` 的程序化返回按当前页面回退到了首页，造成：

```text
homeRemainedAfterDetailBack=false
homeBackHandled=false
```

030 v4 已改用确定超过详情阈值的多段长文本，并在打开详情前强制断言：

```text
detailFixtureEligible=true
detailFixtureLongItemCount>=1
detailFixtureButtonCount>=1
```

## 3. 前置条件

正式运行目录必须已经安装：

```text
moduleSetVersion=20260722.24
sourceRef=agent/initialize-project-skeleton
```

运行探测前先停止正式 ClipHub 实例：

```text
probes/cliphub_stop_formal.js
```

探测会创建隔离运行目录：

```text
shortx.getShortXDir()/ClipHubProbe030/
```

完成后自动停止隔离实例并删除该目录，不修改正式数据库。

## 4. 运行文件

在 ShortX 中执行：

```text
probes/cliphub_system_navigation_probe_030.js
```

固定入口会实时读取开发分支中的：

```text
probes/cliphub_system_navigation_probe_030_impl.js
```

探测输出保存到：

```text
shortx.getShortXDir()/ClipHub/probes/
cliphub_system_navigation_probe_030_时间戳.json
```

本轮返回结果应显示：

```text
probeVersion=4
moduleSetVersion=20260722.24
```

4. 前台 Root Task 基线包名已捕获；
5. 长文本详情测试夹具满足详情入口条件；
6. 详情页返回后关闭详情并恢复首页；

## 6. 必须人工验证

自动调用返回分发器不能代替真实系统手势。探测通过后继续人工操作。

### 6.1 首页

1. 打开全局剪贴板首页；
2. 从屏幕左侧或右侧执行系统侧滑返回；
3. 确认全部 ClipHub UI 消失；
4. 确认原应用仍可正常触摸。

### 6.2 次级页面

分别打开：

- 详情；
- 编辑；
- 筛选；
- 标签管理。

每个页面执行一次侧滑返回，确认只关闭当前最上层页面并回到首页。首页再侧滑一次才退出全部 UI。

### 6.3 三键导航

切换到三键导航后重复首页和次级页面测试，返回顺序必须与侧滑返回一致。

### 6.4 最近任务

1. 打开首页或任意次级页面；
2. 从底部上拉进入最近任务；
3. 确认所有 ClipHub 窗口和遮暗层立即消失；
4. 返回原应用后确认没有透明遮罩；
5. 再次打开 ClipHub，确认从首页开始。

### 6.5 后台监听

在 ClipHub UI 已通过返回或最近任务关闭后：

1. 在其他应用复制一段新文本；
2. 再次打开 ClipHub；
3. 确认该文本已进入历史记录。

## 7. 当前边界

- 探测运行期间会停止隔离实例的剪贴板监听；
- 正式功能不会因 UI 隐藏而停止剪贴板监听；
- ColorOS 或其他定制系统可能调整返回回调和最近任务时序；
- 探测 030 v4 与人工手势均通过前，不将阶段 2D 标记为完成。
