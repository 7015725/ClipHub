# ClipHub 系统手势与后台退出探测 030 说明

## 1. 目标

探测模块集 `20260722.24` 的必要功能：

- Android 系统侧滑返回；
- Android 13 `OnBackInvokedCallback`；
- Android 14 `OnBackAnimationCallback`；
- 三键返回后备链；
- 次级页面返回首页；
- 首页返回关闭全部 ClipHub UI；
- 底部上拉进入最近任务或切换后台后关闭全部 UI；
- UI 关闭后后台实例、数据库和剪贴板监听继续运行；
- 关闭后清理残留 WindowManager View 和系统返回回调引用。

## 2. 历次探测结论

### 030 v2：发现返回去重和回调清理问题

模块集 `20260722.23` 出现：

```text

editorClosedAfterBack=false
homeBackHandled=true
homeClosedAfterBack=false
duplicateBackCount=2
registeredRootCount=5
```

原因：返回分发器只使用全局 `180ms` 时间锁，不区分页面；统一隐藏链也未完全清理失效窗口的返回注册项。

模块集 `20260722.24` 已修正：

- 真实系统返回使用统一 `system` 签名去重；
- 程序化返回按页面所有者去重；
- 关闭链扫描并移除残留 ClipHub WindowManager View；
- 注销失效的系统返回、焦点和附着监听；
- `Navigation.getState()` 自动剔除已分离窗口。

### 030 v3：发现详情测试夹具不足

该次 `ok=false` 属于探测数据问题，不是业务实现回退。

## 6. 尚未完成的人工验证

自动调用返回分发器不能代替真实系统手势。仍需完成以下真机操作。

### 6.1 首页侧滑

1. 启动正式 ClipHub；
2. 从屏幕左侧或右侧执行系统侧滑返回；
3. 确认全部 ClipHub UI 消失；
4. 确认原应用仍可正常触摸。

### 6.2 次级页面侧滑

分别打开：

- 详情；
- 编辑；
- 筛选；
- 标签管理。

每个页面执行一次真实侧滑返回，确认只关闭当前最上层页面并回到首页。首页再次侧滑才退出全部 UI。

### 6.3 三键导航

切换到三键导航后重复首页和次级页面测试，返回顺序必须与侧滑返回一致。

### 6.4 最近任务

1. 打开首页或任意次级页面；
2. 从底部上拉进入最近任务；
3. 确认所有 ClipHub 窗口和遮暗层消失；
4. 返回原应用后确认没有透明遮罩；
5. 再次打开 ClipHub，确认从首页开始。

### 6.5 后台监听

在 ClipHub UI 已关闭后：

1. 在其他应用复制一段新文本；
2. 再次打开 ClipHub；
3. 确认该文本已进入历史记录。
