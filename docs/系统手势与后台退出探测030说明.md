# ClipHub 系统手势与后台退出探测 030 说明

## 1. 目标

探测模块集 `20260722.23` 新增的必要功能：

- Android 系统侧滑返回；
- Android 13 `OnBackInvokedCallback`；
- Android 14 `OnBackAnimationCallback`；
- 三键返回后备链；
- 次级页面返回首页；
- 首页返回关闭全部 ClipHub UI；
- 底部上拉进入最近任务或切换后台后关闭全部 UI；
- UI 关闭后后台实例、数据库和剪贴板监听继续运行。

## 2. 前置条件

正式运行目录必须已经安装：

```text
moduleSetVersion=20260722.23
sourceRef=agent/initialize-project-skeleton
```

运行探测前先停止正式 ClipHub 实例：

```text
probes/cliphub_stop_formal.js
```

探测会创建隔离运行目录：

```text
shortx.getShortXDir()/ClipHubProbe030/
```

完成后自动停止隔离实例并删除该目录，不修改正式数据库。

## 3. 运行文件

在 ShortX 中执行：

```text
probes/cliphub_system_navigation_probe_030.js
```

该文件是固定入口加载器，会读取同一开发分支中的：

```text
probes/cliphub_system_navigation_probe_030_impl.js
```

探测输出保存到：

```text
shortx.getShortXDir()/ClipHub/probes/
cliphub_system_navigation_probe_030_时间戳.json
```

## 4. 自动检查

自动探测验证：

1. `ClipHub.Navigation` 已初始化；
2. Android 13/14 返回回调已注册，或旧版本后备链可用；
3. 主窗口从不可聚焦状态安全升级为可接收系统返回；
4. 详情页返回后关闭详情并恢复首页；
5. 编辑页返回后关闭编辑窗口；
6. 筛选页返回后关闭筛选窗口；
7. 首页返回后关闭主列表；
8. 首页关闭后 `ClipHub.App` 仍在运行；
9. 合成最近任务隐藏会关闭全部 UI；
10. 合成最近任务隐藏后后台实例仍在运行；
11. 隔离实例停止后运行锁已释放。

自动部分通过时：

```json
{
  "ok": true
}
```

## 5. 必须人工验证

自动调用返回分发器不能代替真实系统手势。探测通过后继续人工操作：

### 5.1 首页

1. 打开全局剪贴板首页；
2. 从屏幕左侧或右侧执行系统侧滑返回；
3. 确认全部 ClipHub UI 消失；
4. 确认原应用仍可正常触摸。

### 5.2 次级页面

分别打开：

- 详情；
- 编辑；
- 筛选；
- 标签管理。

每个页面执行一次侧滑返回，确认只关闭当前最上层页面并回到首页。首页再侧滑一次才退出全部 UI。

### 5.3 三键导航

切换到三键导航后重复首页和次级页面测试，返回顺序必须与侧滑返回一致。

### 5.4 最近任务

1. 打开首页或任意次级页面；
2. 从底部上拉进入最近任务；
3. 确认所有 ClipHub 窗口和遮暗层立即消失；
4. 返回原应用后确认没有透明遮罩；
5. 再次打开 ClipHub，确认从首页开始。

### 5.5 后台监听

在 ClipHub UI 已通过返回或最近任务关闭后：

1. 在其他应用复制一段新文本；
2. 再次打开 ClipHub；
3. 确认该文本已进入历史记录。

## 6. 结果回传

需要回传：

- 探测脚本直接返回的完整 JSON；
- `cliphub_system_navigation_probe_030_*.json`；
- 首页侧滑、次级页面侧滑和底部上拉的实际表现；
- 出现异常时的 `shortx.getShortXDir()/ClipHub/logs/cliphub.log`。

## 7. 当前边界

- 探测运行期间会停止隔离实例的剪贴板监听，避免测试数据受真实剪贴板事件干扰；
- 正式功能不会因 UI 隐藏而停止剪贴板监听；
- 进入后台的识别使用窗口焦点、聚焦 Root Task、activity type 和前台包名延迟复核；
- ColorOS 或其他定制系统可能调整返回回调和最近任务时序，因此真实手势复核是发布阻断条件；
- 探测 030 与人工手势均通过前，不将阶段 2D 标记为完成。
