# ClipHub 最小历史列表探测 013

## 1. 目的

`cliphub_list_probe_013.js` 验证阶段 2C 的最小剪贴板历史列表：

- WindowManager 内容宿主替换；
- 原生 ScrollView 卡片列表；
- 空状态；
- ClipboardManager 新记录事件自动刷新；
- 单击卡片复制原文；
- 自身写入回环抑制；
- `closeAfterCopy` 设置；
- 主线程渲染和点击；
- 完整资源清理。

## 2. 当前版本

```text
moduleSetVersion=20260721.12
probeVersion=1
```

## 3. 实现边界

本阶段只实现：

- 最近记录列表，默认最多 20 条；
- 最多三行的文本预览；
- 来源应用、复制次数和时间元数据；
- 敏感记录正文遮罩；
- 单击复制；
- 空状态；
- 剪贴板新增或合并后的自动刷新；
- 亮暗色基础适配。

暂不实现：

- 搜索和筛选；
- 删除、撤销和编辑；
- 置顶和拖动排序；
- 标签；
- 长文本详情；
- 复杂视觉动画。

## 4. 隔离边界

013 使用：

```text
shortx.getShortXDir()/ClipHubProbe013/
```

不会把三条测试文本写入正式数据库。

探测会：

1. 停止正式 ClipHub；
2. 启动隔离实例；
3. 打开空列表；
4. 通过系统 ClipboardManager 写入三条不同文本；
5. 验证 EventBus 驱动列表自动刷新；
6. 通过真实 View `performClick()` 触发卡片点击监听；
7. 验证自身写入不新增数据库记录；
8. 验证“复制后关闭”；
9. 软删除隔离记录并验证空状态；
10. 恢复原剪贴板和来源；
11. 恢复正式实例并删除隔离目录。

报告不包含原剪贴板正文和三条测试正文。

## 5. 执行顺序

1. 执行 `cliphub_stop_formal.js`；
2. 执行完整 `ClipHub.js`，确认同步到 `20260721.12`；
3. 执行 `cliphub_list_probe_013.js`。

入口成功后不要再次停止正式实例，013 会自行完成停止和恢复。

## 6. 可见现象

执行期间会短暂出现 ClipHub 列表窗口：

- 初始显示空状态；
- 随后出现三条测试卡片；
- 第一次自动点击后窗口保持显示；
- 第二次自动点击启用“复制后关闭”，窗口关闭；
- 再次打开后显示空状态；
- 最终自动关闭。

## 9. 失败定位

- `emptyInitial=false`：隔离目录清理失败或空状态渲染错误；
- `eventInsertedCount<3`：ClipboardManager 回调丢失或记录失败；
- `eventRefreshObserved=false`：List 未订阅 `clipboard_added` / `clipboard_merged`；
- `renderThreadName!=main`：View 构建线程边界错误；
- `firstCopyMatched=false`：卡片绑定内容或写剪贴板失败；
- `ownWriteCountStable=false`：单击复制产生回环记录；
- `closeAfterCopyApplied=false`：设置未读取或关闭失败；
- `emptyAfterDelete=false`：空状态刷新错误；
- `clipboardRestored=false`：原剪贴板恢复失败。
