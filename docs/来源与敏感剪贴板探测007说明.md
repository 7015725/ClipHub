# ClipHub 来源与敏感剪贴板探测 007

## 1. 目的

`cliphub_source_probe_007.js` 用于验证 Android 14 / system_server / uid 1000 环境中的以下能力：

1. `ClipboardManager.getPrimaryClipSource()` 是否可访问；
2. 是否可以得到当前剪贴板来源包名；
3. 来源包名能否解析为应用名称和 UID；
4. `ClipboardManager.setPrimaryClipAsPackage()` 是否可访问；
5. `ClipDescription.EXTRA_IS_SENSITIVE` 是否可以写入和读取；
6. 探测后能否恢复原剪贴板内容和原来源包名；
7. 正式 ClipHub 能否在探测后重新启动。

007 只做能力探测，不修改正式数据库 schema，也不把来源策略直接写入正式模块。

## 2. 前置条件

必须满足：

```text
moduleSetVersion=20260721.7
controlTransport=dynamic_broadcast_token
```

正式 ClipHub 必须正在运行。

007 会先通过随机令牌控制端点停止正式实例，确认：

- 控制确认由 Android `main` 线程写入；
- 正式数据库关闭；
- 正式文件锁释放；
- 控制端点删除。

随后才会读取和修改系统剪贴板。

## 3. 隐私边界

报告不会输出原剪贴板正文。

报告可能包含：

- 原剪贴板来源包名；
- 来源应用名称；
- 来源应用 UID；
- MIME 类型；
- 是否带敏感标记；
- 分类状态和时间戳。

探测会临时写入一条固定测试文本，并添加：

```text
android.content.extra.IS_SENSITIVE=true
```

探测结束后会：

1. 恢复原 ClipData；
2. 如果原来源包名可读取且隐藏写入接口可用，精确恢复原来源包名；
3. 重新启动正式 ClipHub。

## 4. 执行方法

同步分支：

```sh

python3 scripts/validate_es5.py .
```

确认探测版本：

```sh
grep -n "probeVersion" probes/cliphub_source_probe_007.js
```

应看到：

```text

```

复制：

```sh
cp probes/cliphub_source_probe_007.js \
  "/storage/emulated/0/Download/ClipHub_来源与敏感剪贴板探测_007.js"
```

在 ShortX 中新建 Rhino JavaScript 单次任务，粘贴完整代码并执行。

执行前不要停止正式 ClipHub，也不要再次运行入口。

## 5. 关键字段

预期：

```text
ok=true
moduleSetVersion=20260721.7
formalControl.ok=true
formalControl.ack.threadName=main
sourcePermissionGranted=true
sourceMethodAvailable=true
sourceReadOk=true
setSourceMethodAvailable=true
setSourceMethodOk=true
testSourceMatched=true
sensitiveDetected=true

clipboardRestored=true
originalSourceRestoredExact=true
formalRestart.ok=true
error=null
```

## 6. 结果解释

### sourceMethodAvailable=false

设备框架没有向 Rhino 暴露隐藏的来源读取接口，需要改用 Binder / `IClipboard` 反射或其他来源推断方案。

### sourceReadOk=false

接口存在，但权限、Hidden API 或服务调用失败。查看对应 `error`。

### setSourceMethodAvailable=false

可以读取来源，但不能精确指定来源。正式功能仍可记录来源，但探测恢复原来源时只能恢复 ClipData 内容。

### testSourceMatched=false

测试剪贴板的来源包名与 ShortX 当前 `opPackageName` 不一致，需要分析 OEM ClipboardService 行为。

### sensitiveDetected=false

说明当前系统没有按标准 extras 返回敏感标记，正式策略需要增加兼容读取或内容规则兜底。

### originalSourceRestoredExact=false

原剪贴板内容已恢复，但来源包名没有精确恢复。本字段为严格隐私边界，因此会导致 `ok=false`。
