# ClipHub 统一视觉与自适应布局探测 022

## 目的

探测阶段 3D 的自动化部分，验证当前模块集在不同主题和请求尺寸下不会发生生命周期、附着、安全区或资源清理回归。

本探测不判断主观视觉质量，最终仍需要截图人工复核。

## 版本

```text
moduleSetVersion=20260722.19
probeVersion=1
```

022 不修改生产模块，也不更新 manifest。

## 自动覆盖范围

### 数据样例

隔离数据库写入五条布局样例：

- 短文本；
- 长文本；
- 敏感长文本；
- 带两个标签的记录；
- 置顶记录。

同时使用较长来源名称和较长标签名称，覆盖省略与按钮密度场景。

### 亮色窄窗口

```text
themeMode=light
requestedWidthDp=280
requestedHeightDp=360
```

验证：

- 主窗口附着；
- 主窗口位于安全区内；
- 实际宽高不超过安全区；
- 五条记录、操作按钮、详情入口、排序手柄、标签摘要和敏感遮罩存在；
- 列表在主线程渲染；
- 搜索与筛选窗口可聚焦并可关闭；
- 新增编辑窗口可聚焦并可关闭；
- 标签管理窗口可聚焦并可关闭；
- 长文本详情可滚动、可选择并可关闭；
- 所有次级窗口均完成清理。

### 暗色宽窗口

```text
themeMode=dark
requestedWidthDp=420
requestedHeightDp=620
```

执行与亮色相同的完整链路。宽度或高度超过当前安全区时，由 Window 模块自动夹紧。

### 重启边界

- 五条隔离记录跨重启存在；
- 暗色设置跨重启存在；
- 主窗口、筛选、编辑、标签和详情窗口重启后均保持关闭；
- 数据库关闭；
- 正式实例恢复；
- 隔离目录删除。

## 执行方式

当前正式实例已经运行 `.19`，无需重新运行入口。

同步仓库：

```sh
cd ~/projects/github/ClipHub
git fetch origin
git switch agent/initialize-project-skeleton
git pull --ff-only
python3 scripts/validate_es5.py .
```

确认：

```sh
grep -n "probeVersion" probes/cliphub_visual_layout_probe_022.js
grep -n "REQUIRED_SET" probes/cliphub_visual_layout_probe_022.js
```

复制：

```sh
cp probes/cliphub_visual_layout_probe_022.js \
  "/storage/emulated/0/Download/ClipHub_统一视觉与自适应布局探测_022.js"
```

然后直接执行探测任务。探测会自行停止并恢复正式实例。

## 自动探测的限制

022 能确认控件存在、窗口可用、尺寸被安全区约束、主题值生效和资源被清理，但不能可靠判断：

- 按钮是否视觉拥挤；
- 字体与间距是否协调；
- 文本是否存在肉眼可见的截断；
- 亮暗色对比度是否舒适；
- 卡片层级、圆角和留白是否统一。

因此 022 通过后需要单独运行视觉样例并提交亮色、暗色、筛选、编辑、标签和详情截图，再进行实际 UI 调整。
