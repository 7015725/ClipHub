# ClipHub 新首页 UI 校正探测 034

## 1. 背景

探测 033 已确认第一轮首页实现的模块加载、WindowManager 生命周期、系统返回注册、正式实例停止与恢复、隔离数据库和基础 UI 结构均正常，但真机截图显示面板尺寸、卡片密度、来源图标、更多按钮、选择语义和底部操作栏仍未达到目标。

因此模块集 `20260722.27` 将 `ch_09_list.js` 升级为 v11，并由探测 034 验证第二轮校正结果。

## 2. 第二轮校正内容

### 2.1 面板尺寸

旧调用仍传入 `340 × 500/560dp` 时，首页会根据当前屏幕自动调整到接近：

```text
widthDp≈390
heightDp≈720
```

实际尺寸仍受系统安全区域限制。首页遮暗值调整为：

```text
dimAmount=0.44
```

### 2.2 来源应用图标

优先使用：

```text
PackageManager.getApplicationIcon(source_package)
```

加载失败时才回退到网址、文本、电话、邮箱或代码类型符号。

### 2.3 卡片布局

- 缩小卡片上下内边距和卡片间距；
- 时间位于右上；
- 星标与更多按钮位于右下并横向排列；
- 普通模式不为选择圆点预留空间；
- 管理模式或长按选中后才显示选择区域；
- 普通点击卡片进入详情；
- 更多按钮继续负责标签入口和拖动排序。

### 2.4 选择与复制边界

新增：

```text
performSelectClick(index)
performCardOpenClick(index)
enterSelection(id)
leaveSelection()
```

保留旧接口：

```text
performItemClick(index)
```

旧接口继续表示程序化复制，但选中操作不再修改系统剪贴板。

### 2.5 底部操作栏

- 中央新增按钮使用独立圆形紫色强调；
- 删除按钮不再默认显示红色激活背景；
- 未选中时，置顶、编辑、删除和翻译降低透明度；
- 选择记录后恢复正常可用视觉。

### 4.3 截图结论

截图确认第二轮校正达到首页阶段视觉基线：

- 面板接近全宽且位于底部安全区域；
- 6 条样例可在同一页面内展示，列表密度显著改善；
- 6 个来源应用图标均成功加载；
- 时间、星标和更多按钮没有裁切；
- 选择圆点只在管理状态出现；
- 被选中卡片使用紫色描边与实心勾选；
- 中央新增按钮具有明确主操作层级；
- 选择操作未触发复制。

仍可在最终五页面统一回归阶段继续细调标题图标、线性图标字形和个别间距，但这些不阻塞进入 3D2-2。

## 5. 最终结论

探测 034 通过。

首页 / 列表页第二轮实现可作为阶段 3D2 的当前视觉基线，下一步进入搜索 / 筛选页复刻。

## 6. 边界

- 不修改数据库 schema v2；
- 不修改系统侧滑返回、三键返回和最近任务自动关闭链；
- 不增加自定义底部手势或透明触摸层；
- 首页通过不代表其余四个页面已经完成；
- PR 继续保持 Draft，不合并 `main`。
