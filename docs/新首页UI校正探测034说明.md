# ClipHub 新首页 UI 校正探测 034

## 1. 背景

探测 033 在 Android 14 / ColorOS 真机返回：

```text
ok=true
moduleSetVersion=20260722.26
Theme=2
Window=5
List=10
windowInsideSafeBounds=true
requiredUiPresent=true
error=null
```

这说明第一轮首页实现的模块加载、WindowManager 生命周期、系统返回注册、正式实例停止与恢复、隔离数据库和基础 UI 结构均正常。

但 `ok=true` 只表示结构检查通过。根据用户回传的未裁剪截图，第一轮首页与目标参考图仍有明显差距，不能作为最终视觉基线。

## 2. 探测 033 截图结论

### 2.1 窗口比例

探测 033 实际窗口约为：

```text
widthDp=340
heightDp=560
```

主要问题：

- 面板偏窄；
- 面板偏矮；
- 面板顶部位置偏低；
- 同屏仅能完整展示约 4 张卡片；
- 与参考图中接近全宽、接近大半屏高度的底部浮层比例不一致。

### 2.2 卡片结构

- 来源区域显示的是电话、文字、邮箱等类型符号，而不是来源应用图标；
- 时间、星标和更多按钮采用纵向堆叠，右侧区域过窄；
- 部分卡片的更多按钮不明显或发生裁切；
- 卡片整体高度偏大，信息密度不足；
- 选中勾选区始终占位，使普通浏览状态比参考图更拥挤。

### 2.3 底部操作栏

- 中央“新增”入口强调不足；
- “删除”按钮默认呈红色背景，视觉上像已激活状态；
- 无选中项时其余操作仍显得可立即执行；
- 图标与参考图的线性图标层级仍有差距。

### 2.4 交互问题

探测 033 为制造选中态调用卡片点击，结果同时触发复制：

```text
selectedCount=1
copyCount=1
lastCopiedId=5
```

这说明第一轮代码把“卡片点击复制”和“卡片选中”绑定在一起，不符合新首页目标：

- 普通点击应进入详情；
- 长按或管理模式才进入选中状态；
- 选中不应修改系统剪贴板；
- 旧探测和旧接口仍需保留可程序化复制能力。

## 3. 模块集 20260722.27

本轮只更新：

```text
src/ch_09_list.js
MODULE_VERSION=11
```

Theme、Window、Navigation、RecentsWatch、App、数据库 schema 和其他业务模块保持不变。

### 3.1 尺寸校正

当正式入口或旧调用仍传入 `340 × 500/560dp` 时，首页会依据当前屏幕自动调整到接近：

```text
widthDp≈390
heightDp≈720
```

仍受安全区域和实际屏幕尺寸限制，不会越界。

首页遮暗值调整为：

```text
dimAmount=0.44
```

### 3.2 来源应用图标

优先使用：

```text
PackageManager.getApplicationIcon(source_package)
```

加载失败时才回退到网址、文本、电话、邮箱或代码类型符号。

新增状态字段：

```text
actualSourceAppIconCount
```

### 3.3 卡片校正

- 缩小卡片上下内边距和卡片间距；
- 时间独立位于右上；
- 星标与更多按钮改为右下横向排列；
- 普通模式不再为勾选圆点预留空间；
- 进入管理或长按选中后才显示选择区域；
- 普通点击卡片进入详情；
- 长按卡片进入选择模式；
- 更多按钮仍负责标签入口和拖动排序。

### 3.4 选择与复制边界

新增：

```text
performSelectClick(index)
performCardOpenClick(index)
enterSelection(id)
leaveSelection()
```

保留旧接口：

```text
performItemClick(index)
```

其语义仍为程序化复制，但不进入选中模式。

探测 034 会明确验证：

```text
selectionMode=true
selectedCount=1
copyCount=0
lastCopiedId=null
selectionDidNotCopy=true
```

### 3.5 底部操作栏校正

- 中央新增按钮使用独立圆形紫色强调；
- 删除按钮不再默认使用红色激活背景；
- 未选中记录时，置顶、编辑、删除和翻译降低透明度；
- 选择记录后恢复正常可用视觉。

## 5. 运行步骤

1. 在 Termux 同步 `agent/initialize-project-skeleton`；
2. 运行原有 `ClipHub.js`，同步到模块集 `.27`；
3. 将 `probes/cliphub_new_home_ui_probe_034.js` 的完整内容复制到新的 ShortX JavaScript 任务；
4. 建议任务名称：`ClipHub 新首页UI校正探测034`；
5. 运行探测；
6. Toast 出现“034 首页第二轮校正版 · 请截图”后截取一张未裁剪原始截图；
7. 等待约 18 秒，回传完整 JSON 和截图。

## 7. 边界

- 探测使用隔离数据库，不修改正式剪贴板历史；
- 正式实例原本运行时，探测结束后自动恢复；
- 不修改数据库 schema v2；
- 不修改系统侧滑返回、三键返回和最近任务自动关闭链；
- 不增加自定义底部手势或透明触摸层；
- 本轮仍属于首页视觉校正，不代表其余四个页面已经复刻；
- PR 继续保持 Draft，不合并 `main`。
