2. 标签名称、颜色预览、关联数量和拖动手柄正常；

9. 编辑已有记录与新增记录均支持多标签；
10. Filter Root 卡片使用标签自定义颜色，并继续显示两个标签和 `+N`；

12. 数据库 schema v2、Navigation v3、正式实例恢复和运行锁清理保持不变。

## v2 修正

Editor 的 `openNew(options)` 不接收 `text` 作为初始正文。v1 的新增记录测试虽然传入了 `text`，实际编辑框仍为空，会导致新增记录保存断言误报。

v2 加载器对固定实现源码应用确定性的探测兼容修正：

```text
openNew({ requestKeyboard:false })
→ Editor.setInputText("ClipHub 新记录标签保存测试")
→ 打开标签选择器
→ 完成标签草稿
→ 保存新记录
```

最终结果额外要求：

```text
newEditorTextSet=true
newEditorSave=true
newItemId>0
newItemTags=[tagC,tagA]
```

该修正只影响探测脚本，不修改 Editor v10 或业务模块。
