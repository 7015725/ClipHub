# ClipHub 阶段 1 实现说明

## 1. 本阶段目标

阶段 1 建立可持续开发的运行骨架，并完成首页 / 列表页的第一版视觉基线。

本阶段不接入真实剪切板、SQLite、来源识别和翻译网络请求。页面使用固定演示数据，避免数据链路问题干扰 UI 校正。

## 2. 已实现

### 2.1 测试分支

- 创建 `beta` 分支；
- `main` 保持需求和架构文档状态；
- 后续视觉开发先进入 `beta`。

### 2.2 单入口启动

`ClipHub.js` 负责：

- 建立 `shortx.getShortXDir()/ClipHub-Beta/`；
- 从 GitHub `beta` 分支下载缺失或版本变化的模块；
- 原子安装模块文件；
- 网络失败时使用已有完整模块回退；
- 按固定顺序加载模块；
- 默认执行 `toggle` 命令。

支持命令：

```text
show
close
toggle
status
```

入口默认值：

```javascript
var CLIPHUB_BOOT_COMMAND = "toggle";
```

也可在调用入口前设置：

```javascript
var CLIPHUB_COMMAND = "show";
```

### 2.3 进程级单实例

通过 `java.lang.System.getProperties()` 保存 Java 运行桥：

- `show`；
- `close`；
- `toggle`；
- 当前可见状态。

重复执行入口时优先调用已有窗口，不重新创建第二个 WindowManager 根窗口。

### 2.4 WindowManager 外壳

已建立：

- `TYPE_APPLICATION_OVERLAY`；
- `FLAG_DIM_BEHIND`；
- `SOFT_INPUT_ADJUST_RESIZE`；
- 底部默认位置；
- 屏幕安全区域夹取；
- 顶部手柄拖动；
- Back 键返回或关闭；
- 幂等移除；
- 关闭时取消动画和清理运行桥。

### 2.5 视觉组件

已建立：

- 固定参考图紫色主题；
- 圆角背景工厂；
- TextView 文字样式；
- 自绘图标；
- 图标按钮；
- 标签 Chip；
- 搜索框；
- 应用图标；
- 底部快捷操作；
- 卡片背景。

### 2.6 首页视觉基线

首页包含：

- 拖动手柄；
- 标题栏；
- 搜索框；
- 筛选按钮；
- 新增按钮；
- 数量和排序提示；
- 七条演示卡片；
- 选中状态；
- 收藏状态；
- 底部五项快捷操作。

其他页面当前只建立路由、返回、关闭和占位视觉骨架，下一阶段逐页替换。

## 4. ShortX 测试方法

推荐直接运行探测脚本：

```text
probes/cliphub_stage1_visual_probe_001.js
```

脚本会从 `beta` 分支获取最新 `ClipHub.js` 并打开首页。

测试项目：

1. 首次运行能够自动下载模块；
2. 页面能够正常显示；
3. 顶部手柄可以拖动窗口；
4. 点击搜索、筛选、新增、设置或卡片可以进入占位页面；
5. 占位页面返回正常；
6. Back 键能够返回或关闭；
7. 再次运行入口只切换已有窗口；
8. 关闭后屏幕不存在透明触摸遮罩。

## 5. 已知边界

- 当前只实现亮色参考图模式；
- 卡片使用固定演示数据；
- 搜索框当前是可点击容器，不是真实输入框；
- 拖动排序仅保留交互入口；
- 置顶、收藏、删除和翻译尚未接入数据层；
- 页面位置暂未持久化；
- 来源应用图标取决于设备是否安装对应应用；
- 视觉数值需要根据 Android 14 真机截图继续校正。
