# ClipHub 内容、标签、翻译设置探测 045

探测版本：`v3`。模块集：`20260723.02`。

## v3 修正

探测 v2 的产品功能实际通过，但场景和断言存在三项误差：

- 首批页面只会显示 URL 卡片的两个标签，不应要求累计三个标签；
- 场景 1 截图前不应先执行加载更多、长按和翻译；
- 翻译弹窗附着后应等待 `running=false` 的稳定失败终态；
- 有道配置必须在打开设置页之前写入，使页面选择态与 SQLite 配置一致。

## 验证内容

- URL、电话等输入统一按普通文本处理；
- 内容类型筛选完全停用；
- 默认首页首批加载 20 条并可滚动；
- 场景 1 截图时保持 `selectionMode=false`；
- 加载更多后显示全部 28 条夹具记录；
- 长按后点击翻译可打开独立弹窗；
- 空翻译凭据进入稳定失败状态，不得导致 `system_server` 重启；
- 关闭翻译弹窗后返回唯一 Filter Root；
- 设置页打开时 `selectedEngine=youdao` 且 `configuredEngine=youdao`；
- 百度和有道凭据继续结构化保存于 SQLite；
- 标签管理、schema v2、Navigation v3、正式实例恢复和运行锁清理保持正常。
