# ClipHub 真实手指排序探测 021

## 目的

在探测 020 已验证排序数据模型与合成触控后，使用真实手指完成一次卡片排序，确认：

- 用户手指事件真实经过 `↕` 手柄；
- ScrollView 不会抢走已经开始的排序手势；
- 主悬浮窗不会跟随卡片排序移动；
- 排序结果与 SQLite `manual_order` 一致；
- 排序结果跨重启保持；
- 排序事件不包含正文。

## 版本

```text
moduleSetVersion=20260722.19
probeVersion=1
```

本阶段不更新生产模块，也不更新 manifest。020 通过后正式实例已经运行 `.19`，只需拉取并复制 021 探测文件。

## 操作方法

执行探测后会显示三条隔离测试记录，顶部记录内容为：

```text
请按住本条左侧 ↕，拖到最下方后松手
```

在 25 秒内完成：

1. 按住顶部记录左侧的 `↕`；
2. 向下拖过中间记录；
3. 拖到最下方记录位置；
4. 松开手指。

不要拖动 ClipHub 顶部标题栏。检测到一次真实排序提交后，窗口会自动关闭并完成重启持久化检查。

## 真实触控判定

探测不会调用：

```text
List.performReorder()
List.performReorderHandleDrag()
```

通过条件同时要求：

```text
reorderDragCommitCount > 0
reorderSyntheticCount = 0
列表顺序发生变化
```

因此合成排序或直接调用 Repository 不会被认定为真实手指通过。

## 隔离边界

探测使用：

```text
shortx.getShortXDir()/ClipHubProbe021/
```

不会修改正式数据库，不读取或替换系统剪贴板。探测结束后会：

- 关闭隔离窗口；
- 停止隔离 App；
- 关闭隔离数据库；
- 恢复正式 `.19` 实例；
- 删除隔离目录。

## 超时结果

25 秒内没有完成有效排序时，通常会看到：

```text
ok=false
userDragDetected=false
interactionWaitMs≈25000
```

这种结果不表示数据库或正式实例损坏。探测仍会恢复正式实例并清理隔离目录，可直接重新执行 021。
