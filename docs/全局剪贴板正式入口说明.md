# ClipHub 全局剪贴板正式入口

## 目的

ClipHub 后台实例负责持续监听和保存系统剪贴板，但不会自动显示历史列表。本阶段增加正式控制入口，让其他 ShortX 任务可以安全打开、关闭或切换全局剪贴板窗口。

## 模块版本

```text
moduleSetVersion=20260722.22
ch_15_app MODULE_VERSION=5
control endpoint schemaVersion=2
```

## 支持命令

```text
show
hide
toggle
status
stop
```

### show

关闭筛选、编辑、标签或详情等次级窗口，然后打开或刷新全局剪贴板列表。

默认参数：

```text
limit=100
widthDp=340
heightDp=500
```

### hide

关闭所有 ClipHub 用户界面，但后台实例、数据库和剪贴板监听继续运行。

### toggle

只要主列表、详情、编辑或筛选窗口中任意一个正在显示，就关闭全部界面；全部关闭时则打开历史列表。

### status

返回脱敏运行状态：

```text
started
uiVisible
listVisible
windowAttached
detailAttached
editorAttached
filterAttached
itemCount
renderedCount
filterActive
```

不返回剪贴板正文、标签名称、来源名称或搜索条件。

### stop

保持原有语义：停止正式实例、关闭数据库、释放锁并删除控制端点。旧版 `cliphub_stop_formal.js` 继续兼容。

## 用户任务

### 始终打开

```text
tasks/ClipHub_打开全局剪贴板.js
```

用于桌面快捷方式、ToolHub 按钮或明确的“打开”动作。即使当前存在编辑、筛选或详情窗口，也会统一关闭后显示主列表。

### 开关切换

```text
tasks/ClipHub_全局剪贴板开关.js
```

首次执行打开，再次执行关闭。适合悬浮球、快捷设置磁贴、按键或手势触发。

## 运行前提

必须先执行完整 `ClipHub.js`，让后台正式实例保持运行。若后台未运行，入口任务返回：

```text
ok=false
running=false
error=ClipHub 后台未运行，请先执行完整 ClipHub.js
```

## 安全边界

- 动态随机广播 action；
- 192 位随机控制令牌；
- 同时校验运行目录和令牌；
- 每次请求使用独立回执文件；
- UI 控制在 Android main 线程执行；
- 回执不包含正文；
- 控制端点只在全部模块初始化成功后发布；
- 停止实例时端点和令牌立即删除。

## 推荐绑定

首选绑定 `ClipHub_全局剪贴板开关.js`：

- ToolHub 按钮；
- ShortX 桌面快捷方式；
- 悬浮球长按；
- 快捷设置磁贴；
- 音量键组合或边缘手势。

主列表右上角 `×` 只关闭窗口，后台监听不会停止。
