# ClipHub 搜索 / 筛选新 UI 探测 035

## 1. 目标

验证阶段 3D2-2 第一轮搜索 / 筛选页复刻：

- 搜索输入与实时结果；
- 搜索历史；
- 来源应用、内容类型和标签多选；
- 置顶与敏感内容筛选；
- 右侧高级筛选抽屉；
- 重置与应用筛选；
- 来源应用图标和结果卡片；
- WindowManager、安全区、系统返回注册和正式实例恢复边界。

本探测用于结构和真机截图校正，不代表第一轮代码已经达到最终像素一致。

## 4. 场景

### 场景 1：搜索结果页

探测插入 6 条记录，其中 5 条包含 `Android`，然后打开搜索页并搜索：

```text
Android
```

预期：

- 显示 5 条结果；
- 显示来源应用图标；
- 显示搜索历史；
- 高级筛选抽屉未展开；
- 面板宽度不低于 380dp；
- 面板高度不低于 680dp。

Toast：

```text

```

### 场景 2：高级筛选抽屉

探测选择：

```text
来源应用=Chrome 浏览器
内容类型=链接
标签=开发资源
敏感内容=隐藏敏感
```

预期：

- 右侧高级筛选抽屉展开；
- 搜索结果仍保留在左侧背景区域；
- 来源、类型和标签使用多选胶囊；
- 底部显示“重置”和“应用筛选”；
- 当前组合筛选结果只保留 Chrome 的 Android 链接记录。

Toast：


## 5. 运行步骤

1. Termux 同步 `agent/initialize-project-skeleton`；
2. 停止正式实例；
3. 运行原有 `ClipHub.js`，同步模块集 `.28`；
4. 将 `probes/cliphub_search_filter_ui_probe_035.js` 完整复制到新的 ShortX JavaScript 任务；
5. 建议任务名：`ClipHub 搜索筛选UI探测035`；
6. 运行后依次截取两张未裁剪原始截图；
7. 等待探测自动结束；
8. 回传两张截图和完整 JSON。

探测会使用隔离数据库，并在结束后恢复原本正在运行的正式实例。
