# ClipHub 长文本详情探测 019

## 目的

验证模块集 `20260722.18` 的长文本详情链路：

- 仅长文本记录显示“详情”入口；
- 短文本不显示且不能打开详情；
- 详情使用独立原生 `TYPE_APPLICATION_OVERLAY` 窗口；
- 完整正文在可滚动、可选择的 TextView 中显示；
- 主列表敏感记录继续遮罩；
- 用户主动进入详情后才显示敏感正文；
- 详情复制保留敏感标记；
- 详情可以跳转到现有编辑器；
- 关闭按钮和 App.stop 均完整清理详情窗口；
- 重启后不会自动恢复详情或编辑窗口。

## 版本

```text
moduleSetVersion=20260722.18
probeVersion=1
```

## 长文本判定

记录满足任一条件时显示“详情”按钮：

```text
正文长度 > 180 字符
正文行数 >= 4 行
```

该阈值只影响详情入口，不截断数据库正文，也不改变卡片单击复制行为。

## UI 边界

- 主悬浮窗仍由 `ch_08_window.js` 管理；
- 详情窗口由 List 模块独立创建；
- 不使用全屏遮罩；
- 不修改主窗口位置、折叠或拖动逻辑；
- 详情窗口提供“复制”“编辑”“关闭”；
- 详情正文支持系统文本选择；
- 详情打开期间主列表保持存在；
- 从详情进入编辑时，先移除详情窗口，再打开现有编辑器。

## 敏感内容边界

主列表仍显示：

```text
敏感内容
```

只有用户主动点击该记录的“详情”按钮后，详情窗口才显示完整正文。详情复制时向 `ClipHub.Clipboard.writeText()` 传递 `sensitive=true`。

## 探测隔离

019 使用：

```text
shortx.getShortXDir()/ClipHubProbe019/
```

探测通过临时替换隔离实例中的 `ClipHub.Clipboard.writeText()` 捕获复制参数，因此：

- 不读取用户系统剪贴板；
- 不替换用户系统剪贴板；
- 不修改正式数据库；
- 报告不包含测试正文。

## 执行顺序

1. 执行 `cliphub_stop_formal.js`；
2. 执行完整 `ClipHub.js`，确认同步到 `20260722.18`；
3. 执行 `cliphub_long_detail_probe_019.js`。

018 最后恢复的是 `.17` 正式实例，因此必须先完成停止和入口更新。
