# ClipHub 删除与撤销探测 014

## 1. 目的

`cliphub_delete_undo_probe_014.js` 验证阶段 3A 的删除与撤销基础链路：

- 列表卡片删除按钮；
- SQLite 软删除；
- 最近一次删除撤销；
- 删除与恢复事件；
- 主线程 View 点击；
- 删除状态重启持久化；
- 撤销状态不跨重启保留；
- WindowManager 与数据库资源清理。

## 2. 当前版本

```text
moduleSetVersion=20260721.13
probeVersion=1
```

## 3. 实现边界

本阶段采用软删除：

```text
deleted_at=<删除时间>
```

删除后记录不会出现在普通历史列表，但正文和其他字段仍保留在数据库中，便于撤销或后续回收站功能使用。

撤销边界：

- 只保留最近一次删除；
- 撤销状态只保存在当前进程内存；
- 不在撤销状态中保存正文；
- 重启后删除状态继续存在；
- 重启后撤销入口清空；
- 本阶段不执行永久删除。

删除与恢复事件只包含：

```text
id
deletedAt
threadId
threadName
```

事件中不包含剪贴板正文。

## 4. UI 行为

每张记录卡片保留单击复制，并在元数据行右侧增加“删除”按钮。

删除后列表顶部显示：

```text
已删除 1 条记录    撤销
```

点击“撤销”后记录恢复到原排序位置。

## 5. 隔离边界

014 使用：

```text
shortx.getShortXDir()/ClipHubProbe014/
```

探测不会修改正式数据库，也不会写入或替换系统剪贴板。

探测会：

1. 停止正式 ClipHub；
2. 启动隔离实例；
3. 插入三条隔离记录；
4. 显示三张卡片；
5. 自动点击第二张卡片的删除按钮；
6. 验证软删除和撤销入口；
7. 自动点击撤销；
8. 再次删除同一记录；
9. 停止并使用同一隔离数据库重新启动；
10. 验证删除状态保留、撤销状态清空；
11. 恢复正式实例并删除隔离目录。

报告不包含三条测试正文。

## 6. 执行顺序

1. 执行 `cliphub_stop_formal.js`；
2. 执行完整 `ClipHub.js`，确认同步到 `20260721.13`；
3. 执行 `cliphub_delete_undo_probe_014.js`。

入口成功后不要再次停止正式实例，014 会自行完成停止和恢复。

## 7. 可见现象

执行期间会短暂出现 ClipHub 窗口：

- 初始显示三条记录；
- 第二条记录自动消失；
- 顶部出现撤销入口；
- 撤销后记录恢复；
- 同一记录再次删除；
- 重启隔离实例后显示剩余两条记录；
- 最终自动关闭。

## 10. 失败定位

- `firstDeleteHidden=false`：删除按钮、软删除或事件刷新失败；
- `softDeleteStored=false`：`deleted_at` 未正确写入；
- `undoAvailableAfterDelete=false`：撤销栏未构建；
- `deleteThreadName!=main`：删除 View 点击线程边界异常；
- `restoredVisible=false`：恢复后列表未重新显示记录；
- `restoredInDatabase=false`：`deleted_at` 未清空；
- `deletionPersistedAfterRestart=false`：软删除未持久化；
- `undoClearedAfterRestart=false`：临时撤销状态错误跨进程保留；
- `eventRefreshObserved=false`：删除或恢复事件未驱动列表刷新。
