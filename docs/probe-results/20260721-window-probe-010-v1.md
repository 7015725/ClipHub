# ClipHub WindowManager 探测 010 v1 结果

## 结果

010 v1 返回 `ok=false`，但 WindowManager 主体操作均成功：


失败字段只有：

```text
attachedToWindow=false
reopenSucceeded=false
```

## 根因

v1 在 `WindowManager.addView()` 返回后立即调用 `View.isAttachedToWindow()`，随后连续执行移动、折叠和关闭。

在当前 system_server / ShortX 环境中，`addView()` 已成功接受窗口，`updateViewLayout()` 和 `removeViewImmediate()` 也能正常执行，但 View 的首帧 attach 尚未来得及由主线程完成，因此即时读取 `isAttachedToWindow()` 返回 `false`。

二次打开同样被这一时序误判影响。`openCount=2` 且最终移除成功，说明第二次 `addView()` 本身没有失败。

## 修正

010 升级到 v2：

- 首次打开后最多等待 1500 ms；
- 每 25 ms 读取一次真实 `isAttachedToWindow()`；
- 只有观察到 `attached=true && attachedToWindow=true` 才继续；
- 二次打开执行同样等待；
- 分别输出 `firstAttachWaitMs` 和 `reopenAttachWaitMs`；
- 窗口真实附着后额外保留 100 ms 再关闭。

运行模块未修改，模块集仍为 `20260721.10`。
