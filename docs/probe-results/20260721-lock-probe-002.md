## 结论

探测通过。

- 两个独立 ShortX 任务运行在同一个 `system_server` 进程；
- 两个任务使用不同的 `DefaultDispatcher` 工作线程；
- 持锁任务按预期保持约 12 秒；
- 竞争任务在 2 毫秒内被阻止；
- 同一 JVM 中的竞争表现为 `OverlappingFileLockException`；
- `FileChannel` 文件锁适合作为 ClipHub 当前环境的单实例边界；
- 正式代码仍需将 `OverlappingFileLockException` 和 `tryLock() == null`
  统一解释为“已有实例运行”。

## 后续边界

本次结果没有验证：

- `system_server` 被强制终止后的恢复流程；
- 文件系统异常或描述符耗尽；
- UI WindowManager 生命周期；
- ClipboardManager 监听器注销。

这些边界分别在后续稳定性探测中处理。
