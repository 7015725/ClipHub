# ClipHub 剪贴板探测 005

## 1. 目的

005 使用真实 `ch_04_clipboard.js` 验证 Android 14 / ShortX / Rhino ES5 环境中的 ClipboardManager 数据链路：

- 注册与注销 `OnPrimaryClipChangedListener`；
- 读取纯文本 ClipData；
- 记录回调线程；
- ClipHub 主动写入后的自身回环抑制；
- 外部文本复制入库；
- 同一内容短时间连续复制合并；
- `copy_count` 更新；
- 停止后数据库关闭和监听器移除；
- 探测结束后恢复原剪贴板；
- 正式 ClipHub 实例重新启动。

## 2. 数据隔离

探测数据库位于：

```text
shortx.getShortXDir()/ClipHubProbe005/data/cliphub.db
```

不会写入正式数据库：

```text
shortx.getShortXDir()/ClipHub/data/cliphub.db
```

探测开始时会暂停正式 ClipHub 实例，结束时恢复原剪贴板并重新启动正式实例。

报告不会写入原剪贴板正文。

## 3. 更新模块

当前入口版本 4 在 ClipHub 已运行时会返回 `reused=true`，不会检查远程 manifest。因此测试模块集 `20260721.5` 前需要先停止当前实例。

在 ShortX 中建立一个临时 JavaScript 任务，执行：

```javascript
(function (global) {
    try {
        if (global.ClipHub && global.ClipHub.App) {
            return JSON.stringify(global.ClipHub.App.stop());
        }
        return JSON.stringify({ok: true, stopped: true, alreadyStopped: true});
    } catch (error) {
        return JSON.stringify({ok: false, stopped: false, error: String(error)});
    }
}((function () { return this; }())));
```

预期：

```json
{"ok":true,"stopped":true}
```

然后重新执行 ShortX 中保存的完整 `ClipHub.js` 入口。预期同步到：

```text
moduleSetVersion = 20260721.5
downloadedCount = 15
transport = raw
```

## 6. 事件边界

Android 的 `OnPrimaryClipChangedListener` 可能因文本分类状态变化而对同一份 ClipData 再次回调。因此模块使用：

```text
内容 SHA-256
+ callbackDedupMs
+ mergeWindowMs
+ ownWrite hash / expiresAt
```

共同判断：

- 同一系统回调的重复通知；
- 用户短时间再次复制相同文本；
- ClipHub 自己写入剪贴板产生的回调。
