# 阶段 3D2 综合收口探测 051 v2 修正说明

## 结论

051 v1 的 `ok=false` 来自探测夹具与既有产品边界冲突，不是正式模块回归。

v1 同时观察到：

```text
contentTypeOptionCount=0
typeFilter=false
tagFilter=true
applyFilter=true
lastResultCount=1
error=null
lastCheckpoint=finished
```

Repository 当前按阶段 3D2-4 已确认的边界处理内容类型：

- `insertItem()` 统一写入 `content_type=text`；
- `listContentTypeOptions()` 返回空数组；
- Filter Root 不展示内容类型筛选项；
- 内容类型设置已停用。

051 v1 却调用 `performTypeClick("url")`，并要求筛选条件包含 `url`，因此该断言必然失败。

## v2 修正

v2 不修改任何正式模块，只修正综合探测：

1. 验证 `Filter.getContentTypeOptions().length === 0`；
2. 不再点击已停用的 URL 类型选项；
3. 使用关键词 `Android` 与标签 `要点` 完成组合筛选；
4. 要求结果窗口仍为唯一一条记录；
5. 其余 Editor、Settings、Navigation、hide/show、stop 和清理断言保持不变。
