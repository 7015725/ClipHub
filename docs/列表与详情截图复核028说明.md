# ClipHub 列表与详情截图复核 028

## 目的

对阶段 3D1 第二部分进行人工视觉复核，只检查新版主列表卡片与长文本详情窗口。

本探测不修改生产模块、manifest、正式数据库或系统剪贴板。

## 版本

```text
moduleSetVersion=20260722.21
probeVersion=1
sceneDurationMs=10000
```

## 场景

### 1. 暗色窄列表

```text
widthDp=300
heightDp=500
```

重点检查：

- 来源元数据独立一行后是否可读；
- 五个操作按钮是否越界或过密；
- 排序手柄宽度是否合理；
- 标签摘要与敏感遮罩的层级；
- 卡片垂直间距与整体信息密度。

### 2. 亮色宽列表

```text
widthDp=400
heightDp=560
```

重点检查：

- 亮色卡片边界和对比度；
- 长来源名称的省略位置；
- 长文本、短文本、敏感和置顶记录的高度节奏；
- 操作区在较宽窗口中的对齐方式。

### 3. 暗色长文本详情

重点检查：

- 主列表外壳是否完全隐藏；
- 是否只剩一个关闭入口；
- 背景是否不透明；
- 正文字号、行距和阅读密度；
- 正文框、窗口边框和按钮层级是否过重；
- 编辑、复制和关闭按钮的间距。

## 数据样例

隔离数据库写入四条记录：

- 普通短文本；
- 普通长文本；
- 敏感多行长文本；
- 带两个标签的置顶记录。

剪贴板监听会在隔离实例启动后停止，避免轮播期间混入系统剪贴板记录。
