# ClipHub 搜索 / 筛选第二轮校正探测 036

## 1. 目标

验证阶段 3D2-2 第二轮校正，重点覆盖探测 035 暴露的测试夹具误报和高级筛选抽屉视觉问题。

## 3. 第二轮改动

### Filter v7

- 搜索页样式标识升级为 `reference_search_v2`；
- 高级筛选来源、类型、标签改为多行自动换行；
- 高级抽屉增加独立关键词输入框；
- 增加“最新优先 / 置顶优先 / 来源应用”排序状态；
- 排序范围限定为 Filter 当前结果窗口，Repository 查询语义不变；
- 高级抽屉固定为约 `238 × 540dp`，底部“重置 / 应用筛选”保持固定；
- 顶部筛选按钮展开后仍显示“筛选”；
- `showPanel()` 重新读取 Settings 中的搜索历史，修复模块初始化顺序导致的正式重启恢复问题；
- 增加 `handleBack()`：高级抽屉先返回搜索页，搜索页再返回首页。

### Navigation v3 / Translation v4

Navigation 仍由 `ch_12_translation.js` 内嵌提供。本轮只把 Filter 返回交给 `Filter.handleBack()`；最近任务、后台切换和全局隐藏仍调用完整 `closePanel()`，不会改变已通过的系统返回和 UI 清理链。

### 探测 035 夹具修正

无关记录包名由：

```text
com.google.android.apps.docs
```

改为：

```text
com.google.docs.editor
```

原包名包含 `android`，按 Repository 既有语义应当命中，因此 035 的第 6 条结果属于测试夹具问题，不是搜索实现错误。
