# ClipHub 剪贴板压力探测 006

## 1. 目的

`cliphub_pressure_probe_006.js` 同时验证：

1. 正式 ClipHub 实例的跨 ShortX 任务停止控制；
2. 控制确认文件和正式文件锁释放；
3. 100 条不同文本连续复制；
4. 多 item 文本读取；
5. 空白、非文本和超长 ClipData 忽略；
6. 停止清理、剪贴板恢复和正式实例重启。

## 2. 前置条件

必须满足：

- ShortX 入口已同步到模块集 `20260721.6` 或更高；
- 正式 ClipHub 当前正在运行；
- 入口返回中存在：

```text
controlAction=com.cliphub.runtime.CONTROL
```

006 会严格要求正式实例正在持有：

```text
shortx.getShortXDir()/ClipHub/data/cliphub.lock
```

如果正式实例未运行，006 会直接失败，不执行压力测试，也不会继续修改剪贴板。

## 3. 隔离边界

压力测试使用：

```text
shortx.getShortXDir()/ClipHubProbe006/
```

不会把 100 条压力测试文本写入正式：

```text
shortx.getShortXDir()/ClipHub/data/cliphub.db
```

测试完成后会：

- 停止隔离实例；
- 关闭隔离数据库；
- 恢复执行前的原剪贴板；
- 重新启动正式 ClipHub；
- 删除 `ClipHubProbe006` 隔离目录。

报告不包含原剪贴板正文。

## 4. 执行方法

同步分支：

```sh
cd ~/projects/github/ClipHub
git fetch origin
git switch agent/initialize-project-skeleton
git pull --ff-only
```

确认脚本版本：

```sh
grep -n "probeVersion" probes/cliphub_pressure_probe_006.js
```

复制：

```sh
cp probes/cliphub_pressure_probe_006.js \
  "/storage/emulated/0/Download/ClipHub_剪贴板压力探测_006.js"
```

在 ShortX 中新建 Rhino JavaScript 单次任务，粘贴完整代码并执行。

## 5. 测试内容

### 5.1 跨任务停止控制

006 发送：

```text
action=com.cliphub.runtime.CONTROL
command=stop
runtimeDir=<正式 ClipHub 运行目录>
requestId=<本次探测 ID>
```

正式实例应：

- 在 `main` 线程接收；
- 注销控制接收器；
- 注销 ClipboardManager 监听器；
- 关闭数据库；
- 释放文件锁；
- 写入 requestId 对应确认文件。

### 5.2 100 条不同文本

006 以约 25 ms 间隔写入 100 条不同文本，要求隔离数据库最终准确保存 100 条记录，且无错误事件。

### 5.3 ClipData 边界

随后验证：

- 两个文本 item 合并为带换行的一条记录；
- 纯空白文本被忽略；
- `text/uri-list` 非文本内容被忽略；
- 超过探测配置 `maxChars=4096` 的文本被忽略；
- 无效 ClipData 不增加数据库记录。

## 8. 失败处理

- `formalInitiallyRunning=false`：先执行正式入口，再运行 006；
- `formalAckReceived=false`：检查控制接收器注册和广播 package；
- `formalAckThreadName!=main`：控制接收器线程边界异常；
- `uniqueCount<100`：检查监听回调丢失、数据库写入错误或等待时间；
- `uniqueCount>100`：检查同一 clip 重复回调抑制；
- `invalidCountStable=false`：检查空白、非文本或超长内容过滤；
- `formalRestart` 失败：检查文件锁、接收器注销和数据库关闭顺序。
