# ClipHub 视觉截图人工复核 023

## 目的

022 已确认窗口尺寸、安全区、附着、主题状态和资源生命周期正常。023 用隔离样例依次展示九个实际 UI 场景，供人工截图复核：

- 按钮密度；
- 卡片间距；
- 字体和层级；
- 长来源、长标签和时间元数据截断；
- 窄窗口溢出；
- 空状态与撤销条；
- 亮色和暗色对比度；
- 输入法出现后的编辑窗口布局；
- 长文本详情窗口的留白与操作区。

## 版本

```text
moduleSetVersion=20260722.19

sceneDurationMs=9000
```

023 不修改生产模块、不更新 manifest、不修改正式数据库，也不会读取或替换系统剪贴板。

## 场景顺序

每个场景停留 9 秒，总时长约 81 秒。主窗口标题栏显示当前序号和场景名称。

### 1/9 亮色窄列表

```text
themeMode=light
requestedWidthDp=280
requestedHeightDp=360
```

重点观察：

- 顶部“新增/筛选”是否拥挤；
- 卡片操作按钮是否截断或挤压；
- 左侧排序手柄宽度；
- 长来源元数据省略；
- 敏感记录遮罩；
- 标签摘要与按钮行的垂直间距。

### 2/9 暗色宽列表

```text
themeMode=dark
requestedWidthDp=420
requestedHeightDp=620
```

重点观察亮暗色层级、卡片边界、按钮底色和正文对比度。

### 3/9 搜索与筛选窗口

不主动弹出输入法。重点观察搜索框、来源/类型/标签芯片、底部重置按钮和滚动区域。

### 4/9 筛选摘要与单条结果

自动应用“长文本视觉”关键词。重点观察筛选摘要是否抢占空间、单条卡片是否过高。

### 5/9 筛选空状态

自动应用不存在的关键词。重点观察空状态居中、上下留白和筛选标题。

### 6/9 删除后的撤销条

自动软删除短文本样例。重点观察撤销条高度、文字与按钮对齐，以及与首张卡片的间距。场景结束后自动撤销。

### 7/9 新增编辑窗口

会请求系统输入法。重点观察标题、输入框、保存/取消按钮、输入法遮挡和窗口高度。

### 8/9 标签管理窗口

会请求系统输入法。重点观察长标签名称、绑定按钮、删除按钮和横向/纵向密度。

### 9/9 长文本详情窗口

重点观察正文滚动区、复制/编辑/关闭按钮、正文与操作区留白。
