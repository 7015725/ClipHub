# ClipHub 视觉截图人工复核 023

## 目的

022 已确认窗口尺寸、安全区、附着、主题状态和资源生命周期正常。023 用隔离样例依次展示九个实际 UI 场景，供人工截图复核：

- 按钮密度；
- 卡片间距；
- 字体和层级；
- 长来源、长标签和时间元数据截断；
- 窄窗口溢出；
- 空状态与撤销条；
- 亮色和暗色对比度；
- 输入法出现后的编辑窗口布局；
- 长文本详情窗口的留白与操作区。

## v2 修正

023 v1 存在两个探测问题：

1. `Window.setStatusText()` 会清空 List 自定义内容，导致列表截图为空；
2. 81 秒探测期间隔离 Clipboard 监听器继续接收系统剪贴板，记录数由 5 增长为 7，使暗色列表和撤销条的严格计数失败。

v2 已调整为：

- 不再调用 `Window.setStatusText()`；
- 场景序号使用短暂 Toast 提示；
- 隔离实例启动后立即调用 `ClipHub.Clipboard.stop()`；
- 要求 `clipboardListenerStopped=true`；
- 列表场景验证 `Window.contentMode=custom`。

该修正只影响探测文件，模块集仍为 `.19`。

## 场景顺序

每个场景停留 9 秒，总时长约 81 秒。场景切换时会短暂显示 Toast。

1. 亮色窄列表；
2. 暗色宽列表；
3. 搜索与筛选窗口；
4. 筛选摘要与单条结果；
5. 筛选空状态；
6. 删除后的撤销条；
7. 新增编辑窗口；
8. 标签管理窗口；
9. 长文本详情窗口。
