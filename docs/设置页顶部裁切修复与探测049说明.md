# ClipHub 设置页顶部裁切修复与探测 049

## 现象

标签管理场景中，设置页顶部出现白色内容横向切入：面板圆角仍在，但滚动内容从旧偏移位置开始显示，标题栏和前置分组被截在视口外。

## 根因

`rebuildTagPage()` 调用 `buildPage()` 替换 `ScrollView` 内容后：

1. 原 `ScrollView.scrollY` 没有复位；
2. 随后使用 `mainHandler.post()` 调用 `scrollToSection("tags")`；
3. 新内容尚未完成测量和布局时，`tagsSectionView.getTop()` 可能仍为 0 或旧值；
4. 滚动位置因此保留旧偏移，形成顶部硬切；
5. `panelRoot` 仅设置圆角背景，没有启用 `clipToOutline`，滚动子内容不会严格遵守外层圆角轮廓。

## 正式修复目标

升级：

```text
Settings 7 → Settings 8
moduleSetVersion 20260723.05 → 20260723.06
```

保持：

```text
entryVersion=4
databaseSchemaVersion=2
Navigation=3
Editor=11
15 模块结构
ClipHub.js 唯一入口
```

修复内容：

1. 每次 `buildPage()` 替换内容后立即执行 `scrollRoot.scrollTo(0, 0)`；
2. 标签变更后的自动定位改为绑定当前 `ScrollView` 的双阶段 `View.post()`；
3. 执行前检查设置页仍 attached，且 `ScrollView` 未被替换；
4. 面板根容器在 API 21+ 启用 `setClipToOutline(true)`；
5. 状态新增滚动复位、请求、应用、取消和标签区视口顶部位置指标；
6. 不修改标签数据库事务、拖动排序、翻译配置、窗口尺寸公式或返回逻辑。
