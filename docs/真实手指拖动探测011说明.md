# ClipHub 真实手指拖动探测 011

## 1. 目的

`cliphub_touch_drag_probe_011.js` 用于补齐 010 自动生命周期探测无法覆盖的真实触摸边界：

- 用户真实按住标题栏；
- Android `MotionEvent` 进入标题栏触摸监听器；
- `dragMoveCount` 增加；
- 窗口最终坐标发生变化；
- 更新操作仍运行在 Android `main` 线程；
- 拖动后窗口仍位于安全区域内。

011 不调用 `moveTo()` 或 `moveBy()` 制造移动。只有标题栏触摸监听器中的拖动路径会增加 `dragMoveCount`，因此可以区分程序移动和真实手指拖动。

## 2. 当前版本

```text
moduleSetVersion=20260721.10
probeVersion=1
```

本探测没有修改正式模块，不需要重新下载 15 个模块。

## 3. 执行时的可见操作

探测窗口出现后会显示：

```text
请按住标题栏拖动窗口
检测到真实拖动后会自动关闭
```

请在 15 秒内：

1. 按住顶部 `ClipHub` 标题区域；
2. 将窗口拖动一段明显距离；
3. 松开手指。

检测到有效拖动后，窗口会显示“已检测到真实拖动”，随后自动关闭。

请不要点击右上角关闭按钮。若 15 秒内没有检测到拖动，探测会失败但仍会清理隔离实例并恢复正式 ClipHub。

## 4. 隔离边界

011 使用：

```text
shortx.getShortXDir()/ClipHubProbe011/
```

探测会：

- 停止正式 ClipHub；
- 启动隔离实例；
- 创建一个真实 WindowManager 窗口；
- 等待用户拖动；
- 移除窗口并关闭隔离数据库；
- 恢复正式实例；
- 删除隔离目录。

不会修改正式数据库或系统剪贴板。

## 5. 执行方法

同步分支：

```sh
cd ~/projects/github/ClipHub

git pull --ff-only
```

确认版本：

```sh
grep -n "probeVersion" probes/cliphub_touch_drag_probe_011.js
```

```text

```

复制：

```sh
cp probes/cliphub_touch_drag_probe_011.js \
  "/storage/emulated/0/Download/ClipHub_真实手指拖动探测_011.js"
```

010 v2 最后已经恢复正式实例，所以直接执行 011。不要先运行停止脚本，也不要重新执行入口。

## 6. 通过条件

```text
ok=true
moduleSetVersion=20260721.10
formalControl.ok=true
formalControl.ack.threadName=main
isolatedStart.ok=true
windowOpen.ok=true
attachedToWindow=true
userDragDetected=true
dragMoveCount>0
positionChanged=true
finalInsideBounds=true
updateThreadName=main
closeResult.ok=true
detachedAfterClose=true
removeThreadName=main
appStopped=true
databaseClosed=true
formalRestart.ok=true
cleanup=true
error=null
```

通常还会看到：

```text
interactionWaitMs<15000
initialPosition.x != finalPosition.x
或
initialPosition.y != finalPosition.y
```

## 7. 失败定位

- `attachedToWindow=false`：窗口没有在 1.5 秒内完成 attach；
- `dragMoveCount=0`：触摸没有进入标题栏拖动路径；
- `dragMoveCount>0` 但 `positionChanged=false`：移动距离不足 8 px，或坐标被边界夹紧；
- `finalInsideBounds=false`：拖动安全区夹紧异常；
- `updateThreadName!=main`：WindowManager 更新线程边界异常；
- `interactionWaitMs>=15000`：未在操作窗口内完成有效拖动。
