# ClipHub 正式版后台入口与单一开关收口

## 目标

正式版只保留两个 ShortX 任务：

```text
ClipHub.js
tasks/ClipHub_全局剪贴板开关.js
```

职责边界：

- `ClipHub.js`：启动并维护后台实例、数据库、剪贴板监听、模块同步和控制端点；启动完成后默认不显示 UI。
- `ClipHub_全局剪贴板开关.js`：唯一用户入口；当前有任意 ClipHub UI 时关闭全部 UI，无 UI 时打开唯一 Filter Root。

删除：

```text
tasks/ClipHub_打开全局剪贴板.js
```

不新增独立关闭任务。

## 控制端点 schema v3

端点继续使用动态随机 action 和随机 token，并增加版本元数据：

```text
schemaVersion
transport
action
token
runtimeDir
entryVersion
moduleSetVersion
sourceRef
commands
createdAt
```

控制回执同步返回：

```text
endpointSchemaVersion
entryVersion
moduleSetVersion
sourceRef
```

## 单一开关任务

开关任务不再硬编码具体 `moduleSetVersion`，只校验：

- endpoint schema 至少为 3；
- entryVersion 至少为 5；
- moduleSetVersion 非空；
- sourceRef 非空；
- transport 和 runtimeDir 匹配；
- commands 包含 `toggle`。

因此后续只更新模块时，不需要同步修改 ShortX 开关任务。

默认运行目录仍为：

```text
shortx.getShortXDir()/ClipHub
```

探测可通过内部选项指定隔离运行目录：

```javascript
var ClipHubControlOptions = {
    runtimeName: "ClipHubProbe052"
};
```

正式用户无需配置该选项。

## 安全边界

- 保留运行锁，禁止多个后台实例同时运行；
- 保留动态广播 action 和随机 token；
- 保留独立回执文件；
- UI 控制继续在 Android main 线程执行；
- `toggle/hide` 只关闭 UI，不停止数据库、监听和后台实例；
- `stop` 仍负责完整停止、关闭数据库、释放锁和删除控制端点；
- Rhino ES5 only；
- 不修改 Repository、Filter、Editor、Settings、Navigation 或数据库 schema。
