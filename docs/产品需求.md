# ClipHub 产品需求

## 1. 文档状态

- 阶段：持续迭代需求
- 适用项目：ClipHub
- 目标平台：Android 14 / ShortX / Rhino ES5
- 当前结论：用于约束首个可发布版本；标记为“必须实现”的项目属于发布阻断条件

## 2. 产品目标

ClipHub 需要解决以下问题：

1. 系统剪贴板只能方便地访问当前内容，历史内容查找困难；
2. 大量复制内容缺少来源、类型和分类信息；
3. 网址、电话、命令和普通文本混在一起，操作效率低；
4. 在其他应用中复制内容后，需要快速查看、编辑、翻译或再次复制；
5. 传统全屏管理页面打断当前应用操作，需要一个可控的悬浮窗口。

## 3. 首版范围

### 3.1 必须实现

- 监听并保存文本剪贴板；
- 显示复制时间；
- 尽可能记录来源应用包名与应用名；
- 对重复内容去重；
- 悬浮窗显示、拖动、收起和关闭；
- 剪贴板列表；
- 单击复制；
- 新增、编辑、删除；
- 置顶；
- 拖动排序；
- 关键词搜索；
- 按来源应用过滤；
- 按文本、网址、电话过滤；
- 自定义标签；
- 本地 SQLite 持久化；
- 亮色与暗色模式；
- 支持 Android 系统返回手势、三键返回键和预测性返回；
- 次级页面返回上一层，首页返回时关闭全部 ClipHub UI；
- 用户从底部上拉进入最近任务或切到后台时，立即关闭全部 ClipHub UI；
- 系统手势关闭 UI 后继续保持剪贴板监听和后台实例运行。

### 3.2 第二阶段

- 翻译；
- 批量操作；
- 自动过期；
- 敏感内容保护；
- 规则化自动分类；
- 导入和导出；
- 数据库备份与恢复。

### 3.3 暂不进入首版

- 云端同步；
- 多设备实时同步；
- 图片剪贴板；
- 富文本完整格式还原；
- 跨用户或工作资料空间读取；
- 未经用户配置的自动网络上传。

## 4. 用户操作流程

### 4.1 复制内容

1. 用户在任意应用复制文本；
2. ClipHub 收到剪贴板变化事件；
3. 读取纯文本；
4. 计算规范化摘要；
5. 判断是否为自身写入或短时间重复；
6. 识别来源应用；
7. 判断内容类型；
8. 保存记录；
9. 已打开列表时局部刷新。

### 4.2 再次复制

1. 用户打开 ClipHub；
2. 搜索或过滤记录；
3. 点击目标记录；
4. ClipHub 将原始文本写回系统剪贴板；
5. 当前悬浮窗保持、收起或关闭，由设置决定；
6. 自身监听器忽略本次回写形成的重复事件。

### 4.3 编辑

1. 用户点击“编辑”；
2. 打开编辑层；
3. 修改文本或标签；
4. 保存时更新 `updated_at`；
5. 不修改原始创建时间和来源记录；
6. 编辑后的内容是否重新分类，由设置决定。

### 4.4 删除

- 默认单条删除需要短确认或撤销入口；
- 批量删除必须显示数量；
- 删除标签不能连带删除剪贴板记录；
- 数据库可先采用软删除，为撤销操作保留空间。

### 4.5 系统导航与 UI 退出

#### 系统返回

返回优先级固定为：

```text
筛选 / 编辑 / 详情 / 标签等次级页面
                ↓
关闭当前次级页面并返回首页
                ↓
首页再次返回
                ↓
关闭全部 ClipHub UI
```

要求：

- Android 侧滑返回、三键返回键和预测性返回必须产生一致结果；
- 不允许使用覆盖屏幕边缘的透明触摸层伪造返回手势；
- 首页返回只关闭 UI，不停止剪贴板监听、不关闭数据库、不停止后台实例；
- 快速连续返回不得重复移除同一个 WindowManager View；
- 关闭后不得残留遮暗层、透明遮罩或不可触摸区域。

#### 进入最近任务或后台

用户从底部上拉进入最近任务、切换应用或使 ClipHub 所在前台交互失效时：

1. 关闭筛选、编辑、详情、标签等次级窗口；
2. 关闭主列表窗口；
3. 移除全部 ClipHub 遮暗层和辅助窗口；
4. 保留剪贴板监听、SQLite、控制入口和后台实例；
5. 下次打开时从首页开始，不恢复到上一次次级页面。

编辑中的未保存内容不得因进入后台而自动写入正式记录；系统切后台流程不得弹出阻塞系统手势的确认窗口。

## 5. 搜索与过滤

### 5.1 搜索字段

首版搜索：

- 原始文本；
- 来源应用名称；
- 来源包名；
- 标签名称。

### 5.2 组合规则

不同过滤维度使用 AND：

```text
关键词 AND 来源应用 AND 内容类型 AND 标签
```

同一维度多选使用 OR：

```text
类型 = 网址 OR 电话
```

### 5.3 空状态

需要区分：

- 尚无剪贴板记录；
- 当前搜索无结果；
- 当前过滤条件无结果；
- 数据加载失败。

## 6. 内容类型

### 6.1 普通文本

不满足其他明确规则时归类为普通文本。

### 6.2 网址

要求：

- 支持 `http://` 和 `https://`；
- 识别前后标点但不修改原文；
- 一条文本中包含网址时，可以记录“包含网址”；
- 首版不要求完整解析所有 URI Scheme。

### 6.3 电话号码

要求：

- 支持常见空格和连字符；
- 识别结果必须保留原始显示；
- 拨号操作前交由系统拨号界面确认；
- 避免把长数字、订单号和时间戳轻易误判为电话。

## 7. 来源应用

建议字段：

- 包名；
- 应用显示名；
- UID（可选）；
- 捕获时的前台包名；
- 来源可信度。

来源应用存在无法确认的情况，UI 必须允许显示：

```text
来源未知
```

不能为了补齐来源而阻塞剪贴板保存。

## 8. 自定义标签

标签字段：

- 唯一 ID；
- 名称；
- 颜色；
- 排序；
- 创建时间；
- 更新时间。

约束：

- 标签名称去除首尾空格；
- 空名称不允许保存；
- 同名规则需要统一，建议首版不区分大小写；
- 删除标签时仅删除关联关系；
- 一条记录可绑定多个标签。

## 9. 悬浮窗要求

- 可拖动；
- 不越出安全显示区域；
- 屏幕旋转或尺寸变化后重新夹取位置；
- 支持置顶状态；
- 支持收起为紧凑入口；
- 关闭窗口不等于停止剪贴板监听；
- 停止服务需要独立操作；
- 编辑框打开软键盘时避免窗口被完全遮挡；
- 必须接入系统返回语义，不得仅依赖页面内关闭按钮；
- 系统进入最近任务或后台时必须关闭全部 UI；
- 所有返回、后台切换、按钮关闭和控制命令必须进入同一套幂等 UI 清理链。

## 10. 隐私与安全

- 默认仅本地保存；
- 日志不输出完整剪贴板正文；
- 可配置忽略指定应用；
- 可配置最大保存数量；
- 可配置自动清理时长；
- 对疑似密码、验证码等内容预留不保存或快速过期策略；
- 导出数据必须由用户主动触发。

## 11. 首版验收条件

首版至少满足：

- 连续复制 100 条文本不崩溃；
- 相同文本连续复制不会无限重复；
- ClipHub 自己执行复制不会造成监听回环；
- 重启脚本后记录仍然存在；
- 搜索和来源应用过滤结果正确；
- URL 与电话类型能够过滤；
- 标签增删改与关联正常；
- 悬浮窗拖动不触发列表排序；
- 列表排序不拖动整个窗口；
- 亮色与暗色模式下文字可读；
- 次级页面侧滑返回上一层；
- 首页侧滑返回关闭全部 ClipHub UI；
- 三键返回与侧滑返回行为一致；
- 底部上拉进入最近任务后全部 ClipHub UI 已关闭；
- 系统手势关闭 UI 后剪贴板监听仍继续；
- 再次打开时从首页开始；
- WindowManager 页面关闭后没有残留遮罩或不可触摸区域。
