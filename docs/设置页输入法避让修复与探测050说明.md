# ClipHub 设置页输入法避让修复与探测 050

## 现象

设置页中的翻译配置、标签名称和颜色输入框获得焦点后，输入法会覆盖设置面板下半部分。面板保持底部固定高度，当前输入框不会自动进入可见区域。

剪贴板记录编辑页使用的对应机制称为 **Editor IME 避让**，代码位于：

```text
src/ch_10_editor.js
ClipHub.Editor
reference_editor_v5
```

## 根因

Settings v9 只有普通 `EditText`：

- 未设置 `SOFT_INPUT_ADJUST_RESIZE`；
- 未读取 `WindowInsets.Type.ime()`；
- 未检测输入法可见高度；
- 未根据键盘顶部动态修改面板高度、重力和 Y 坐标；
- 未把当前输入框滚动到可见区域；
- 未在关闭页面时取消 IME 轮询和延迟回调。

## 正式修复目标

```text
Settings 9 → Settings 10
moduleSetVersion 20260723.07 → 20260723.08
```

保持：

```text
entryVersion=4
databaseSchemaVersion=2
Editor=11
Navigation=3
15 模块结构
ClipHub.js 唯一入口
```

## 修复内容

1. 设置窗口启用 `SOFT_INPUT_ADJUST_RESIZE`；
2. 使用 Root WindowInsets、InputMethodManager 可见高度和 visible display frame 三路读取 IME 状态；
3. 键盘弹出时将设置面板切换到顶部重力，并缩高到 IME 上方可用区域；
4. 键盘关闭时恢复原高度、底部重力和 10dp 底边距；
5. 所有设置输入框绑定统一焦点监听；
6. 当前输入框获得焦点后自动滚动到面板可见区域；
7. 为 IME 自动滚动增加独立底部 spacer，不覆盖 Settings v9 的标签区 section anchor spacer；
8. 增加 90ms IME polling 和全局布局观察；
9. 页面关闭和 shutdown 时先停止轮询、移除布局观察并使旧延迟任务失效；
10. 延迟回调异常不得逃逸到 `system_server` 主线程。

## 探测 050

ShortX 新建 JS 任务，完整复制：

```text
probes/cliphub_settings_ime_avoidance_probe_050.js
```

探测要求正式实例正在运行 `.08 / Settings 10`。

### 场景 1

聚焦“百度 APP ID”输入框并显示键盘。

验证：

- 面板位于输入法上方；
- 面板高度小于正常高度；
- 当前输入框完整可见；
- `panelGravity=ime_top`；
- `keyboardAvoidanceApplied=true`。

### 场景 2

聚焦标签管理中的“新标签名称”输入框。

验证：

- 页面自动滚动到标签管理区域；
- 输入框完整位于键盘上方；
- `inputAutoScrollCount>=1`；
- 标签区 section anchor 与 IME anchor 独立工作。

### 恢复场景

探测关闭键盘后验证：

- `keyboardVisible=false`；
- `keyboardAvoidanceApplied=false`；
- `panelGravity=bottom`；
- 当前高度恢复正常高度；
- IME spacer 恢复为 0；
- 延迟回调错误为 0。
