# 阶段 3D2 综合收口探测 051

## 目标

探测 051 用于验证阶段 3D2 全局代码审查后的正式基线，不新增业务功能，不替代 047～050 的视觉证据。

固定要求：

```text
moduleSetVersion=20260723.09
Editor=12
Settings=11
Filter=14
Navigation=3
App=8
databaseSchemaVersion=2
```

## 覆盖范围

051 在正式 ClipHub 实例运行时执行以下链路：

1. 通过控制广播隐藏正式实例，不停止正式实例；
2. 启动 `ClipHubProbe051` 隔离实例；
3. 建立三条记录和三个结构化标签；
4. 验证唯一 Filter Root；
5. 验证关键词搜索；
6. 验证高级筛选、标签筛选和抽屉返回；
7. 验证 Editor 空闲 420ms / 活跃 90ms 自适应 IME 轮询；
8. 验证 Editor 输入法避让和关闭恢复；
9. 验证标签选择取消不落库、最终保存才提交标签；
10. 验证 Settings 空闲 420ms / 活跃 90ms 自适应 IME 轮询；
11. 验证 Settings 输入法避让、关闭恢复和返回 Filter Root；
12. 验证根页面返回、重新 show；
13. 停止隔离实例并验证数据库关闭、锁释放和目录清理；
14. 恢复正式实例显示。

本探测无需截图。

## 执行前置条件

先停止旧正式实例，再运行唯一入口 `ClipHub.js`，确认：

```text

```

正式实例必须保持运行。051 只会对正式实例执行 `hide` 和 `show`，不会执行 `stop`。

