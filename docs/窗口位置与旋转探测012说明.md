# ClipHub 窗口位置与旋转探测 012

## 1. 目的

`cliphub_window_persistence_probe_012.js` 用于验证阶段 2B：

- 窗口位置写入 SQLite `settings` 表；
- 停止并重新启动后恢复窗口位置；
- 使用安全区内归一化比例适配屏幕方向和尺寸变化；
- `ComponentCallbacks` 与 `DisplayManager.DisplayListener` 能收到变化；
- 变化后在 Android `main` 线程更新 WindowManager；
- 新安全区内重新夹紧，不遗留屏幕外窗口。

## 2. 当前版本

```text
moduleSetVersion=20260721.11
probeVersion=1
```

## 3. 存储结构

复用现有 SQLite `settings` 表，不升级数据库 schema。

新增设置：

```json
{
  "windowPosition": {
    "xRatio": 0.27,
    "yRatio": 0.62
  }
}
```

比例基于窗口在当前安全区域内可移动的距离计算：

```text
xRatio = (x - safeLeft) / (safeWidth - windowWidth)
yRatio = (y - safeTop) / (safeHeight - windowHeight)
```

数值始终夹在 `0..1`。屏幕方向或尺寸变化后，使用新安全区和新可移动距离重新计算像素坐标。

首次升级时 `windowPosition=null`，继续使用当前默认位置，不会修改正式历史或其他设置。

## 4. 自动持久化边界

- 标题栏真实拖动结束时写入一次 SQLite；
- 拖动过程中的每个 MOVE 事件不会写数据库；
- `moveTo(x, y, {persist:true})` 可显式保存；
- 普通 `moveTo()` 和 `moveBy()` 只移动，不自动保存；
- 配置变化只按已有比例重投影，不重复写数据库。

## 5. 配置变化监听

窗口模块同时注册：

```text
android.content.ComponentCallbacks
android.hardware.display.DisplayManager.DisplayListener
```

收到变化后等待约 180 ms，让 WindowMetrics 和系统栏 Insets 稳定，再执行：

- 重新读取安全区域；
- 按可用空间缩小窗口尺寸；
- 按原位置比例计算新坐标；
- 四边夹紧；
- 主线程 `updateViewLayout()`。

## 6. 隔离边界

012 使用：

```text
shortx.getShortXDir()/ClipHubProbe012/
```

探测写入的位置设置只存在于隔离数据库，不修改正式 `ClipHub/data/cliphub.db`。

探测结束后会：

- 移除测试 WindowManager View；
- 关闭隔离数据库；
- 删除 `ClipHubProbe012`；
- 恢复正式 ClipHub 实例。

## 7. 执行顺序

当前正式实例仍为模块集 `20260721.10`，因此先升级：

1. 执行 `cliphub_stop_formal.js`；
2. 执行完整 `ClipHub.js`；
3. 确认 `moduleSetVersion=20260721.11`；
4. 执行 `cliphub_window_persistence_probe_012.js`；
5. 测试窗口提示后，在 20 秒内将设备旋转到横屏。

入口成功后不要再次执行停止任务，012 会自行停止和恢复正式实例。

## 8. 可见操作

012 首先自动：

1. 打开测试窗口；
2. 将窗口移动到安全区约 `27% / 62%` 位置并保存；
3. 停止隔离实例；
4. 使用同一数据库重新启动；
5. 确认像素坐标和比例恢复。

随后窗口显示：

```text
请将设备旋转到横屏
检测完成后会自动关闭
```

请将设备从竖屏旋转到横屏。检测完成后窗口会自动关闭。测试结束后可自行转回竖屏。

## 10. 常见失败

- `observersRegistered=false`：检查 ComponentCallbacks 或 DisplayListener 注册；
- `positionPersisted=false`：检查 `windowPosition` 标准化和 SQLite 写入；
- `restoredPixelMatch=false`：检查第二次打开是否读取持久设置；
- `rotationDetected=false`：必须旋转到横屏，180 度翻转可能不会改变安全区尺寸；
- `callbackObserved=false`：检查 system_server 上下文的配置与显示监听；
- `boundsRefreshObserved=false`：检查延迟刷新任务是否执行；
- `rotatedRatioPreserved=false`：检查旧安全区比例到新安全区的投影；
- `rotatedInsideBounds=false`：检查窗口尺寸收缩和四边夹紧。
