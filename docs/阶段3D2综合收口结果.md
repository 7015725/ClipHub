## 全局代码审查处理

### Editor v12

- IME 轮询由固定 90ms 改为自适应：键盘显示或输入框有焦点时 90ms，空闲时 420ms；
- `View.post()` 回调纳入 generation、ready、attached 和空上下文保护；
- 延迟回调异常继续由状态计数器收口，不允许逃逸到 system_server 主线程；
- 标签草稿事务、正文保存语义和输入法避让保持不变。

### Settings v11

- IME 轮询由固定 90ms 改为自适应：键盘显示或输入框有焦点时 90ms，空闲时 420ms；
- 输入框自动滚动和标签区定位的 `View.post()` 回调纳入生命周期保护；
- 标签 section anchor 与 IME anchor 保持独立；
- Settings v10 已通过的顶部圆角、标签定位和输入法避让保持不变。

## 探测 051 v1 夹具修正

051 v1 误把已经停用的内容类型筛选作为正式回归条件：

- Repository 当前新增记录的 `content_type` 固定为 `text`；
- `listContentTypeOptions()` 固定返回空数组；
- 因此 URL 类型按钮不存在，`performTypeClick("url")` 必然返回 false。

该失败属于探测夹具错误，不属于 Filter 或 Repository 回归。

051 v2 改为验证：

- 内容类型选项确实为空；
- 关键词 `Android` 与标签 `要点` 组合筛选；
- 应用后唯一结果为 1 条；
- 其他 Editor、Settings、Navigation、控制入口和清理断言不变。
