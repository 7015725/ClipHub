# ClipHub 最近任务时间线探测 031 说明

## 1. 背景

模块集 `20260722.24` 已通过系统返回自动探测 030 v4，并完成真实侧滑人工验证：

- 首页侧滑返回通过；
- 详情侧滑返回通过；
- 编辑侧滑返回通过；
- 筛选侧滑返回通过；
- 标签侧滑返回通过；
- 重新打开从首页开始；
- UI 关闭后剪贴板监听继续入库；
- 不存在透明遮罩或不可触摸区域。

当前唯一失败项：

```text
底部上拉进入最近任务后 ClipHub UI 未自动关闭
```

合成调用 `Navigation.hideUi("probe_recent_tasks")` 已验证能够完整关闭 UI 并清理系统返回回调，因此问题位于“真实最近任务切换信号没有触发统一隐藏链”，而不是隐藏链自身。

## 2. 探测目标

探测 031 不修改业务逻辑，也不主动关闭 ClipHub UI。它在 15 秒内以 120ms 间隔记录真实底部上拉最近任务过程中的状态变化，用于判断：

1. WindowManager Root 是否收到焦点丢失；
2. 某个 ClipHub Root 是否在最近任务界面中仍保持 `hasWindowFocus=true`；
3. `ActivityTaskManager.getFocusedRootTaskInfo()` 是否切换到 HOME、RECENTS、Launcher 或 SystemUI；
4. 前台包名是否离开 ShortX；
5. Navigation 的焦点计数、后台检查计数和隐藏计数是否变化；
6. 最近任务信号出现时 UI 是否仍保持显示。

## 3. 前置条件

正式运行目录必须安装：

```text
moduleSetVersion=20260722.24
sourceRef=agent/initialize-project-skeleton
```

运行前停止正式实例：

```text
probes/cliphub_stop_formal.js
```

探测使用隔离目录：

```text
shortx.getShortXDir()/ClipHubProbe031/
```

探测结束后会停止并删除隔离实例，不修改正式数据库。

## 4. 运行文件

在 ShortX 中新建或覆盖任务，粘贴完整文件：

```text
probes/cliphub_recent_tasks_timeline_probe_031.js
```

加载器固定读取提交：

```text
c9c03c3f6b9cb212d7ed21772b8a2a560798546a
```

对应实现：

```text
probes/cliphub_recent_tasks_timeline_probe_031_impl.js
```

## 5. 操作顺序

1. 运行探测 031；
2. ClipHub 窗口出现后，在 3 秒内从屏幕底部上拉进入最近任务；
3. 在最近任务界面停留至少 3 秒；
4. 返回原应用；
5. 不要手动关闭 ClipHub；
6. 等待约 15 秒，直到 ShortX 返回 JSON。

窗口中会显示同样的操作提示。

## 6. 输出

摘要直接作为 ShortX 任务结果返回。

完整逐样本时间线保存到：

```text
shortx.getShortXDir()/ClipHub/probes/
cliphub_recent_tasks_timeline_probe_031_时间戳.json
```

文件结构：

```text
result
samples[]
```

`result.changeTimeline` 只保存状态发生变化的样本，便于直接回传；`samples[]` 保存全部 120ms 采样数据。

## 8. 判断方式

### A. 最近任务出现，但 ClipHub Root 仍有焦点

```text
recentsSignalObserved=true
clipHubFocusStuckDuringRecentsSignal=true
```

后续修复应弱化 `anyFocused()` 的绝对拦截，并以 ActivityTaskManager 最近任务信号优先触发隐藏。

### B. 焦点丢失，但后台检查没有执行

```text
focusLossObserved=true
backgroundCheckDelta=0
```

后续修复应检查焦点回调是否被后续 `focusGain` 取消，或增加 UI 可见期间的低频状态观察器。

### C. 后台检查执行，但条件没有匹配

```text
backgroundCheckDelta>0
backgroundHideDelta=0
```

后续修复应扩展 ColorOS Launcher、SystemUI、Recents 包名或 activity type 判断。

### D. ActivityTaskManager 始终不变化

```text
recentsSignalObserved=false
focusLossObserved=true
```

后续需要增加另一种系统状态来源，例如 TaskStackListener 或 ColorOS 可用的窗口/任务栈信号。

## 9. 边界

- 探测不会拦截底部系统手势；
- 不创建底部透明触摸层；
- 不修改模块集版本；
- 不主动调用 `hideUi()`；
- 探测期间停止隔离实例的剪贴板监听；
- 探测结果确定信号来源后，再进行最小业务修复；
- PR 继续保持 Draft，不合并 `main`。
