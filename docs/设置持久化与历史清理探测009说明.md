# ClipHub 设置持久化与历史清理探测 009

## 1. 目的

`cliphub_settings_cleanup_probe_009.js` 用于验证模块集 `20260721.9` 的以下能力：

1. 设置通过 SQLite `settings` 表持久化；
2. 设置值经过类型、范围和列表标准化；
3. 敏感策略、来源开关和忽略应用可立即应用到 Clipboard 模块；
4. 重启隔离实例后设置仍然存在；
5. 历史天数清理和数量裁剪正确；
6. 置顶记录不会被自动清理；
7. 第二次启动清理结果稳定，不重复删除；
8. 探测结束后正式 ClipHub 重新启动。

## 2. 当前版本

```text
moduleSetVersion=20260721.9
probeVersion=1
schemaVersion=2
```

本阶段复用现有 `settings` 表，不升级数据库 schema。

## 3. 安全默认值

自动清理默认关闭：

```text
historyLimit=0
autoCleanupDays=0
```

`0` 表示对应规则禁用。

这是有意的兼容边界：旧占位模块虽然声明过 `500/30`，但此前并未真正执行清理。升级后不会因为旧占位默认值而删除正式历史。

用户显式设置非零值后，才执行数量或天数清理。

置顶记录始终保留。

## 4. 持久设置

当前支持：

```text
historyLimit
autoCleanupDays
closeAfterCopy
themeMode
sourceEnabled
sensitivePolicy
ignorePackages
```

其中：

- `historyLimit`：`0-100000`，0 禁用数量裁剪；
- `autoCleanupDays`：`0-3650`，0 禁用过期清理；
- `themeMode`：`system/light/dark`；
- `sensitivePolicy`：`skip/save`；
- `ignorePackages`：去空格、去空项并去重。

## 5. 清理语义

### 天数清理

删除：

```text
is_pinned=0
last_copied_at < 当前时间 - autoCleanupDays
```

### 数量裁剪

按以下顺序保留最新的非置顶记录：

```text
last_copied_at DESC, id DESC
```

超出 `historyLimit` 的非置顶记录永久删除。

设置标签关联使用外键级联同步清理。

### 顺序

一次清理事务中：

1. 先删除过期非置顶记录；
2. 再按数量裁剪剩余非置顶记录。

## 6. 探测隔离边界

009 使用：

```text
shortx.getShortXDir()/ClipHubProbe009/
```

不会把测试设置或测试记录写入正式数据库。

探测会：

1. 停止正式 ClipHub；
2. 启动隔离实例；
3. 写入 7 项设置；
4. 插入 1 条过期置顶、1 条过期非置顶和 5 条近期非置顶记录；
5. 执行 `30 天 + 3 条` 清理；
6. 停止并再次启动隔离实例；
7. 验证设置和清理状态；
8. 删除隔离目录；
9. 重新启动正式 ClipHub。

探测不修改系统剪贴板。

## 7. 执行顺序

当前正式实例是模块集 `20260721.8`，入口版本 4 不会在实例运行中同步模块，因此按以下顺序执行。

### 7.1 同步仓库

```sh
cd ~/projects/github/ClipHub

git fetch origin
git switch agent/initialize-project-skeleton
git pull --ff-only

python3 scripts/validate_es5.py .
```

确认：

```sh
grep -n "moduleSetVersion" module-manifest.json
grep -n "probeVersion" probes/cliphub_settings_cleanup_probe_009.js
```

应看到：

```text
20260721.9
probeVersion: 1
```

### 7.2 复制探测脚本

```sh
cp probes/cliphub_settings_cleanup_probe_009.js \
  "/storage/emulated/0/Download/ClipHub_设置与历史清理探测_009.js"
```

### 7.3 停止正式实例

执行已有：

```text
ClipHub_停止正式实例.js
```

必须确认：

```text
ok=true
initiallyRunning=true
ackReceived=true
ackThreadName=main
lockReleased=true
endpointRemoved=true
```

### 7.4 执行入口

运行现有完整 `ClipHub.js`，确认：

```text
moduleSetVersion=20260721.9
updated=true
downloadedCount=15
```

正式数据库不会因默认设置而自动删除历史，因为清理规则默认均为 0。

### 7.5 执行 009

入口成功后不要再次停止或运行入口，直接执行：

```text
ClipHub_设置与历史清理探测_009.js
```

## 10. 当前边界

- 设置模块已有持久化 API，但尚无设置页 UI；
- `closeAfterCopy` 和 `themeMode` 已持久化，但对应 UI 行为仍待后续模块使用；
- 清理当前为启动时和设置变更时执行，没有定时任务；
- 自动清理是永久删除，不进入软删除区；
- 入口运行中自动停止并升级仍待入口版本 5。
