# ClipHub 拖动排序与触控隔离探测 020

## 目的

验证模块集 `20260722.19` 的第一阶段排序链路：

- 卡片独立拖动手柄；
- 合成 MotionEvent 经过真实 OnTouchListener；
- SQLite `manual_order` 事务化重编号；
- 置顶与普通记录分组隔离；
- 筛选状态下隐藏并禁用排序；
- 卡片单击复制行为保持；
- 卡片排序不触发主窗口标题栏拖动；
- 排序状态跨重启持久化。

## 版本

```text
moduleSetVersion=20260722.19
probeVersion=1
schemaVersion=2
```

本阶段不修改数据库 schema，继续使用现有 `manual_order` 字段。

## 排序规则

1. 只有未启用筛选的完整列表显示 `↕` 排序手柄；
2. 置顶记录与普通记录分别排序；
3. 不允许跨置顶分组拖动；
4. 首次成功排序会对当前分组全部活动记录进行事务化重编号；
5. `manual_order` 使用 1000、2000、3000 等稀疏序号；
6. 新捕获记录默认 `manual_order=0`，因此出现在对应分组最前方；
7. 切换置顶状态时将 `manual_order` 重置为 0，使记录进入新分组前方；
8. 软删除记录不参与排序。

## 触控边界

- 主悬浮窗仍只在标题栏安装拖动监听；
- 排序手柄位于卡片左侧独立区域；
- 手柄按下后调用 `requestDisallowInterceptTouchEvent(true)`；
- 卡片正文、详情、标签、置顶、编辑和删除按钮不接管排序手势；
- 020 比较排序前后的 `Window.dragMoveCount`，确认主窗口没有被移动。

## 隐私边界

排序事件复用 `clipboard_merged`，但只包含：

```text
id
targetId
mutation=manual_order_changed
pinned
placeAfter
threadId
threadName
```

事件不包含剪贴板正文。

020 临时替换隔离实例的 `Clipboard.writeText()` 验证卡片单击，不读取或替换用户系统剪贴板。

## 隔离范围

```text
shortx.getShortXDir()/ClipHubProbe020/
```

不会修改正式数据库。

## 执行顺序

1. 执行 `cliphub_stop_formal.js`；
2. 执行完整 `ClipHub.js`，确认同步到 `20260722.19`；
3. 执行 `cliphub_reorder_probe_020.js`。

019 最后恢复的是 `.18` 正式实例，因此必须先更新模块集。
