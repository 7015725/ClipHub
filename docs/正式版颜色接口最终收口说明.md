# 正式版颜色接口最终收口说明

## 目标

在 ClipHub 合并 `main` 和发布正式版本前，将 Rhino / ColorOS 数值颜色重载风险收口到唯一 Theme 安全桥，并建立可重复执行的发布门禁。

## 参考依据

参考《拾字_文字选择工具.js》和《Rhino JS 修复 ColorOS 软重启的两个方案》中的稳定路径：

- TextView 使用 `ColorStateList` 对象重载；
- GradientDrawable 填充和描边使用 `ColorStateList` 对象重载；
- Drawable tint 使用 `setTintList(ColorStateList)`；
- Paint 使用 `setARGB(a,r,g,b)`；
- 不使用 `ColorStateList.valueOf(int)`；
- 不使用无 mask 的 `RippleDrawable`；
- 仍不稳定时采用普通 Drawable 与 alpha/scale 按压反馈。

参考脚本中残留的以下路径不纳入 ClipHub：

- `GradientDrawable.setStroke(int, int)`；
- `View.setBackgroundColor(int)`；
- `ColorDrawable(int)`；
- 任何失败后回退到数值颜色重载的逻辑。

## `.14` 变更范围

仅修改：

- `src/ch_07_theme.js`；
- `scripts/audit_color_api.py`；
- `module-manifest.json`。

不修改 UI 色值、布局、圆角、业务逻辑、SQLite、输入法避让、导航、入口或开关协议。

## Theme v4

### 四状态 ColorStateList

统一构造以下状态：

1. pressed；
2. focused；
3. selected；
4. default。

四个状态使用同一个颜色，目的不是改变交互色，而是强制 Rhino 选择对象重载。

### 异常收口

所有安全桥：

- 成功返回 `true`；
- 失败返回 `false`；
- 记录失败次数和最后错误；
- 禁止回退到任何 `int` 颜色重载。

新增状态接口：

```javascript
ClipHub.Theme.getColorSafetyState()
```

字段：

```text
applyCount
failureCount
lastError
```

## 发布审计门禁

新增：

```text
--release-strict
```

正式发布要求：

```text
HIGH=0
WARN=0
```

在原有规则外继续拦截：

- `GradientDrawable.setColors(int[])`；
- `Canvas.drawColor(int)`；
- `Paint.setShadowLayer(..., int)`；
- `PaintDrawable(int)`。

## 执行

预览：

```sh
bash tools/apply_release_color_closure.sh
```

正式应用：

```sh
bash tools/apply_release_color_closure.sh --commit
```

脚本会执行：

- 全仓 Rhino ES5 校验；
- 发布级颜色审计；
- manifest SHA 更新；
- `git diff --check`；
- 禁止数值颜色回退检查；
- 失败自动恢复目标文件。

## 发布边界

`.14` 推送后仍不立即合并 `main`。后续顺序：

1. 更新 ShortX 正式后台到 `.14`；
2. 运行发布真机探测；
3. 连续打开、关闭和操作所有主要 UI；
4. 确认 `failureCount=0`、无 `system_server` 重启；
5. 将 `DEFAULT_REF/sourceRef` 切换到 `main`；
6. 更新 PR 描述并转为 Ready；
7. 合并 `main`；
8. 从 `main` 冷启动验证；
9. 创建正式版本标签。
