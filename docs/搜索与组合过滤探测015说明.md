# ClipHub 搜索与组合过滤探测 015

## 1. 目的

`cliphub_filter_probe_015.js` 验证阶段 3B 的搜索与组合过滤基础链路：

- 关键词搜索；
- 来源应用过滤；
- 内容类型过滤；
- 多条件 AND 组合；
- 多来源和多类型 IN 查询；
- LIKE 通配符按字面量转义；
- SQL 参数绑定；
- 删除记录默认排除；
- 置顶过滤；
- 过滤结果同步到当前列表；
- 剪贴板事件到达后重新应用活动过滤；
- 过滤状态重启后清空。

## 2. 当前版本

```text
moduleSetVersion=20260721.14
probeVersion=1
```

## 3. 实现边界

### Repository

`Repository.listItems()` 新增：

```text
keyword
sourcePackages
contentTypes
pinnedOnly
sensitiveOnly
excludeSensitive
```

所有动态值都通过 SQLite 参数绑定传入。来源和类型数组只生成固定数量的 `?` 占位符，不拼接用户输入。

关键词会搜索：

```text
content
source_label
source_package
```

关键词中的以下字符会按字面量处理：

```text
%
_
\
```

### Filter

过滤状态只保存在当前进程内存：

```text
keyword
sourcePackages
contentTypes
pinnedOnly
sensitiveMode
```

重启后恢复为“全部记录”，不写入 SQLite 设置。

当前提供：

```text
set
reset
setKeyword
setSourcePackages
setContentTypes
setPinnedOnly
setSensitiveMode
query
apply
getSourceOptions
getContentTypeOptions
```

`apply()` 会把查询结果交给现有 List 模块，因此列表卡片和空状态会立即更新。

## 4. 当前 UI 边界

本阶段完成过滤引擎和列表状态接入，但还没有加入用户可见的搜索输入框、来源选择器和类型选择器。015 通过后再实现交互控件，避免同时调试查询正确性和输入法/焦点问题。

## 5. 隔离边界

015 使用：

```text
shortx.getShortXDir()/ClipHubProbe015/
```

探测会插入 6 条活动记录、1 条软删除记录，并在事件重应用阶段增加 1 条活动记录。不会修改正式数据库，也不会读写系统剪贴板正文。

## 6. 执行顺序

1. 执行 `cliphub_stop_formal.js`；
2. 执行完整 `ClipHub.js`，确认同步到 `20260721.14`；
3. 执行 `cliphub_filter_probe_015.js`。

入口成功后不要再次停止正式实例，015 会自行停止、恢复并清理。

## 10. 失败定位

- `literalWildcardCount!=1`：LIKE 转义错误；
- `injectionSafe=false`：查询未完全参数绑定或测试数据异常；
- `deletedExcluded=false`：普通列表错误包含软删除记录；
- `eventFilterReapplied=false`：List 的事件刷新覆盖了活动过滤结果；
- `filteredEmptyState=false`：空过滤结果未同步到 UI；
- `filterResetAfterRestart=false`：临时过滤状态错误持久化；
- `renderThreadName!=main`：过滤结果导致 View 在非主线程构建。
