# ClipHub 详情与列表操作区探测 027

## 目的

验证阶段 3D1 第二部分的生产改动：

- 长文本详情改为不透明模态窗口；
- 详情打开时临时隐藏主列表外壳，消除双层容器和重复关闭入口；
- 关闭详情或从详情进入编辑时恢复主列表；
- 卡片元数据独占一行，不再与操作按钮争抢宽度；
- 操作按钮降字号、缩内边距，但保留原入口和行为；
- 排序、复制、标签、置顶、编辑、删除、撤销和敏感内容边界不回归。

## 版本

```text
moduleSetVersion=20260722.21
probeVersion=1
```

本轮只修改：

```text
src/ch_09_list.js
```

数据库 schema 仍为 v2。

## 详情窗口结构

- `TYPE_APPLICATION_OVERLAY`；
- 完全不透明背景；
- `FLAG_DIM_BEHIND`；
- 暗化量约 `0.72`；
- 不包含 `FLAG_NOT_TOUCH_MODAL`；
- 单一 `×` 关闭入口；
- 正文约 `13sp`，常规字重；
- 行距约 `1.10`；
- 编辑、复制和敏感标记行为保持不变。

详情打开时，主列表 Window 会临时关闭。详情关闭时按原宽高重新打开并渲染当前数据；`list_hide`、`shutdown` 和 `copy_close` 不执行恢复，防止停止过程中重新创建窗口。

## 卡片结构

```text
正文或敏感遮罩
#标签摘要
来源 · 次数 · 时间
详情  标签  置顶  编辑  删除
```

短文本卡片不显示“详情”，其他操作不变。

## 隔离范围

探测使用：

```text
shortx.getShortXDir()/ClipHubProbe027/
```

不会修改正式数据库，也不会读取或替换系统剪贴板。探测会临时替换隔离实例中的 `Clipboard.writeText()`，只记录正文长度和敏感标记。

## 执行顺序

当前正式实例仍为 `.20`，需要：

1. 停止正式实例；
2. 执行完整 `ClipHub.js`，同步到 `.21`；
3. 执行 `cliphub_list_detail_visual_probe_027.js`。

## 自动覆盖范围

- 四条卡片渲染；
- 每条卡片独立元数据行和操作行；
- 卡片单击复制；
- 标签窗口入口；
- 编辑窗口入口；
- 置顶写入；
- 删除与撤销；
- 排序手柄合成 MotionEvent；
- 排序不移动主窗口；
- 普通详情复制；
- 敏感详情复制并传递敏感标记；
- 详情模态、暗化、不透明和主列表隐藏；
- 详情关闭后恢复主列表；
- 详情进入编辑后恢复主列表并打开正确记录；
- App.stop 清理仍打开的详情；
- 置顶和记录跨重启保持；
- 重启后详情和编辑窗口保持关闭。
