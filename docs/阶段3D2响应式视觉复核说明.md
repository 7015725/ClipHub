阶段 2D 的系统返回和最近任务退出链已经完成真机验证。阶段 3D2 只处理主列表、详情和窗口尺寸适配，不改变已经通过的返回、后台监听和 UI 清理逻辑。

### 3D2-A：当前屏幕多宽度视觉基线

使用探测 033 自动展示五个场景并人工截图：

### 3D2-B：横屏、分屏和多窗口

在 3D2-A 截图审查完成后，再建立独立探测验证：

- 竖屏切横屏；
- 横屏切回竖屏；
- 分屏尺寸变化；
- 多窗口安全区变化；
- 拖动后位置比例恢复；
- 次级窗口最大宽度与高度。

将该部分独立处理，避免底部上拉进入最近任务时已经通过的自动关闭逻辑干扰视觉基线采集。

## 场景检查重点

### 场景 1：暗色超窄列表

- 标题和关闭按钮不能互相挤压；
- 卡片正文不能覆盖元数据；
- 元数据、标签和操作按钮不能横向溢出；
- 操作按钮允许合理换行，但不能重叠；
- 敏感内容遮罩必须完整；
- 卡片间距不能过松或过密。

### 场景 2：亮色标准列表

- 亮色背景、边框和文字对比度正常；
- 卡片结构与暗色模式一致；
- 标签和置顶状态清晰；
- 操作区对齐统一；
- 滚动区域底部留白合理。

### 场景 3：暗色宽列表

- 宽度增加后内容不应被无意义拉散；
- 正文和元数据仍保持稳定层级；
- 操作区不能出现过大的空白断层；
- 标题区和卡片圆角比例保持一致。

### 场景 4：暗色长文本详情

- 详情必须是不透明模态窗口；
- 主列表不能从背景透出；
- 正文区域可滚动；
- 长文本换行正常；
- 复制、编辑、关闭按钮完整可见；
- 底部按钮区不能遮挡正文；
- 顶部和底部安全区留白合理。

### 场景 5：亮色折叠窗口

- 折叠后只保留标题区域；
- 右下角位置必须位于安全区内；
- 标题、关闭按钮和圆角不能被裁切；
- 窗口外不存在透明遮罩或触摸拦截区。

## 运行步骤

1. 在 Termux 同步分支；
2. 运行一次 `ClipHub.js`，确认模块集为 `.25`；
3. 将 `probes/cliphub_responsive_visual_review_probe_033.js` 完整复制到新的 ShortX JavaScript 任务；
4. 运行探测 033；
5. 每个场景显示约 10 秒；
6. 根据 Toast 中的场景编号分别截图；
7. 等待五个场景全部结束；
8. 回传完整 JSON 和五张原始截图。

建议 ShortX 任务名称：

```text

```

## 正式实例边界

- 探测使用隔离数据库，不修改正式剪贴板历史；
- 正式实例原本在运行时，探测结束后会自动恢复；
- 正式实例原本未运行时，探测结束后保持未运行；
- 探测不会修改 Manifest 或业务模块；
- 不新增自定义手势、触摸遮罩或底部拦截层；
- 视觉修改不得改变阶段 2D 已通过的返回和最近任务退出链。
