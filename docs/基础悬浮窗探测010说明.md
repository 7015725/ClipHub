# ClipHub 基础悬浮窗探测 010

## 1. 目的

`cliphub_window_probe_010.js` 用于验证阶段 2A 的最小 WindowManager 悬浮窗骨架：

- `TYPE_APPLICATION_OVERLAY` 创建；
- 所有 add/update/remove 操作切换到 Android `main` 线程；
- 重复打开复用同一窗口；
- 状态栏、导航栏和刘海安全区域计算；
- 左上、右下边界夹紧；
- 标题栏拖动监听器安装；
- 折叠、展开、固定状态；
- 幂等关闭；
- 关闭后重新打开；
- App 停止时完整释放 WindowManager View。

## 2. 当前版本

```text
moduleSetVersion=20260721.10
probeVersion=1
```

## 3. 实现边界

本阶段只实现 WindowManager 生命周期骨架，不实现：

- 剪贴板历史列表；
- 搜索、标签和编辑器；
- 窗口位置 SQLite 持久化；
- 屏幕旋转监听；
- 输入法和可聚焦编辑框；
- 完整视觉设计。

窗口默认使用不可聚焦、可触摸的 `TYPE_APPLICATION_OVERLAY`，不会抢占当前应用输入焦点。

## 4. 隔离边界

010 使用：

```text
shortx.getShortXDir()/ClipHubProbe010/
```

探测会先停止正式 ClipHub，再在隔离实例中两次创建和移除窗口，最后恢复正式实例。

不会修改正式数据库或系统剪贴板。

## 5. 执行顺序

1. 执行 `cliphub_stop_formal.js`；
2. 执行完整 `ClipHub.js`，确认同步到 `20260721.10`；
3. 执行 `cliphub_window_probe_010.js`。

入口成功后不要再次停止正式实例，010 会自行完成隔离停止和恢复。

## 7. 可见现象

探测执行期间会短暂显示一个极简 ClipHub 窗口，并快速完成：

- 左上角夹紧；
- 右下角夹紧；
- 折叠和展开；
- 关闭；
- 二次打开；
- 最终关闭。

这属于预期探测行为。
