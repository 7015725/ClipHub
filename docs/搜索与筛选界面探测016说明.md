# ClipHub 搜索与筛选界面探测 016

## 1. 目的

`cliphub_filter_ui_probe_016.js` 验证阶段 3B 的用户可见搜索与筛选界面：

- 主列表“筛选”入口；
- 可聚焦的原生 `EditText`；
- 关键词搜索按钮；
- 来源应用横向选择芯片；
- 内容类型横向选择芯片；
- 组合条件同步到主列表；
- 活动条件摘要；
- 一键重置；
- 筛选面板关闭、重开和 App.stop 自动清理。

## 2. 当前版本

```text
moduleSetVersion=20260721.15
probeVersion=1
```

## 3. 窗口边界

主历史列表继续使用原来的非聚焦 WindowManager 窗口，因此标题栏拖动和外部触摸行为不变。

搜索与筛选使用独立的可聚焦窗口：

```text
TYPE_APPLICATION_OVERLAY
FLAG_NOT_TOUCH_MODAL
FLAG_LAYOUT_IN_SCREEN
FLAG_HARDWARE_ACCELERATED
SOFT_INPUT_ADJUST_RESIZE
```

筛选窗口不设置 `FLAG_NOT_FOCUSABLE`，以便 `EditText` 获取焦点并请求系统输入法。关闭筛选面板或停止 App 时使用 `removeViewImmediate()` 清理。

## 4. 数据边界

016 使用隔离目录：

```text
shortx.getShortXDir()/ClipHubProbe016/
```

会直接向隔离 SQLite 插入 6 条测试记录，不读取或修改系统剪贴板正文，也不会修改正式数据库。

测试记录包含：

- 3 个来源应用；
- `text`、`url`、`phone` 三种类型；
- 4 条包含 `alpha` 的记录。

## 5. 执行顺序

1. 执行 `cliphub_stop_formal.js`；
2. 执行完整 `ClipHub.js`，确认同步到 `20260721.15`；
3. 执行 `cliphub_filter_ui_probe_016.js`。

入口成功后不要再次停止正式实例，016 会自行完成停止和恢复。

## 6. 可见过程

执行期间会短暂出现：

1. 含 6 条记录的主列表；
2. 居中的“搜索与筛选”面板并请求输入法；
3. 搜索 `alpha` 后主列表显示 4 条；
4. 选择 Alpha 来源后显示 2 条；
5. 再选择链接类型后显示 1 条；
6. 重置后恢复 6 条；
7. 关闭并重新打开筛选面板；
8. App.stop 自动清理两个窗口。

所有控件操作通过真实 View `performClick()` 执行，不直接调用筛选结果替换接口。

## 9. 失败定位

- `panelFocusable=false`：筛选窗口错误包含 `FLAG_NOT_FOCUSABLE`；
- `inputFocused=false`：EditText 焦点或窗口输入模式异常；
- `sourceOptionsRendered=false`：来源统计查询或芯片构建失败；
- `typeOptionsRendered=false`：内容类型统计或芯片构建失败；
- `searchCount!=4`：搜索按钮未同步输入内容；
- `sourceFilteredCount!=2`：来源芯片未写入筛选状态；
- `combinedFilteredCount!=1`：类型芯片或组合查询异常；
- `listFilterSummaryVisible=false`：主列表活动条件提示未刷新；
- `panelCleanupOnStop=false`：Filter.shutdown 未移除独立窗口；
- `panelRemoveThreadName!=main`：WindowManager 清理线程边界异常。
